| **US-STRI-15c** | การแจ้งเตือน Strike ให้นักศึกษาทราบ | ในฐานะ **นักศึกษา**<br>ฉันต้องการ **รู้ว่าตัวเองมียอด No-show สะสมเท่าไรและใกล้ถูกตัดสิทธิ์แล้วหรือยัง**<br>เพื่อให้ **ปรับพฤติกรรมได้ก่อนถูกบล็อก** | - หน้า Dashboard แสดง Strike Badge (⚠️ No-show: X/3) เมื่อ `noShowCount ≥ 1`<br>- เมื่อถูกบล็อก ปุ่มลงทะเบียนทุกกิจกรรมแสดงสถานะ Disabled พร้อมคำอธิบาย<br>- Live Notification pop-up แจ้งทันทีเมื่อถูกบันทึก No-show ใหม่ | 🏗 อยู่ในช่วงออกแบบ |
| **US-WAIT-16a** | ระบบเข้าคิว Waitlist เมื่อกิจกรรมเต็ม | ในฐานะ **นักศึกษา**<br>ฉันต้องการ **กดจองคิวรอเมื่อกิจกรรมที่ฉันสนใจเต็มแล้ว**<br>เพื่อให้ **มีโอกาสได้เข้าร่วมโดยอัตโนมัติหากมีที่ว่างเกิดขึ้น** | - ปุ่มลงทะเบียนเปลี่ยนเป็น "เข้าคิวรอ (Waitlist)" เมื่อโควต้าเต็ม<br>- เก็บลำดับคิวด้วย `waitlist_position` โดยใช้ Row Lock (`FOR UPDATE`) กัน Race Condition เช่นเดียวกับโควต้าปกติ<br>- นักศึกษาเห็นลำดับคิวปัจจุบันของตัวเองในหน้ากิจกรรม<br>- สามารถถอนตัวออกจาก Waitlist ได้ตราบใดที่กิจกรรมยังไม่ปิดรับ | 🏗 อยู่ในช่วงออกแบบ |
| **US-WAIT-16b** | เลื่อนคิว Waitlist อัตโนมัติเมื่อมีที่ว่าง | ในฐานะ **นักศึกษาที่รออยู่ใน Waitlist**<br>ฉันต้องการ **ได้รับสิทธิ์ลงทะเบียนโดยอัตโนมัติเมื่อมีคนยกเลิก**<br>เพื่อให้ **ไม่ต้องคอยรีเฟรชหน้าเองและไม่พลาดสิทธิ์** | - เมื่อนักศึกษายกเลิกการลงทะเบียน ระบบเลื่อนคนลำดับที่ 1 ใน Waitlist ขึ้นเป็น `registered` ทันที (Atomic)<br>- กระบวนการเลื่อนคิวต้องรัน Row Lock เพื่อป้องกันการเลื่อนซ้ำ<br>- Live Notification แจ้งนักศึกษาที่ได้สิทธิ์ทันทีผ่าน SSE<br>- หากนักศึกษาที่ได้สิทธิ์ไม่ยืนยันภายในเวลาที่กำหนด (config) สิทธิ์จะตกไปยังคนถัดไปใน Waitlist | 🏗 อยู่ในช่วงออกแบบ |
| **US-WAIT-16c** | การจัดการ Waitlist ฝั่ง Admin | ในฐานะ **ผู้จัดกิจกรรม (Organizer)**<br>ฉันต้องการ **ดูรายชื่อคิวรอและจัดการสิทธิ์ Waitlist ด้วยตนเองในกรณีพิเศษ**<br>เพื่อให้ **ควบคุมสถานการณ์ได้เมื่อระบบอัตโนมัติไม่เพียงพอ** | - หน้า Admin > Events > Attendance แสดง Tab แยก "Waitlist" จาก "Registered"<br>- Admin สามารถ Promote นักศึกษาจาก Waitlist ขึ้นเป็น Registered โดยตรง (override โควต้า +1)<br>- Admin สามารถลบนักศึกษาออกจาก Waitlist ได้<br>- จำนวนคนใน Waitlist แสดงบน Event Card ในหน้า Admin Dashboard | 🏗 อยู่ในช่วงออกแบบ |
