# User Story: US-GAME-18b - ผู้เข้าร่วม Join ด้วย QR หรือ Room Code

**Status:** 🏗 Planned (วางแผน)
**Epic:** [Multi-Interactive Game Session (WebRTC/WebSocket)](../01-product-backlog.md#15-multi-interactive-game-session-webrtc--เกมร่วมกันแบบ-real-time-ในงาน)
**Owner:** Developer
**Version:** 1.0 | **Last Updated:** 2026-06-18

---

## 📖 Description
**ในฐานะ** ผู้เข้าร่วมงาน (Student/Guest)  
**ฉันต้องการ** เข้าร่วมห้องเล่นเกมโดยการสแกน QR Code หรือป้อนรหัสห้อง 6 หลักบนอุปกรณ์มือถือ  
**เพื่อให้** เข้าร่วมห้องเกมและตั้งชื่อเล่นสำหรับการเล่นเกมได้อย่างง่ายดายโดยไม่ต้องผ่านกระบวนการลงชื่อเข้าใช้งานระบบหลักที่ซับซ้อน  

---

## ✅ Acceptance Criteria
1. [ ] ผู้เข้าร่วมที่สแกนคิวอาร์โค้ดจะต้องได้รับ Redirect ไปยังหน้าล็อบบี้ร่วมห้องอัตโนมัติพร้อมป้อนรหัสห้องไว้ให้ล่วงหน้า
2. [ ] มีช่องทางให้ป้อนรหัสห้องด้วยตนเอง (สำหรับคนที่พิมพ์คีย์ 6 หลักบนหน้าเว็บหลัก)
3. [ ] ผู้เล่นต้องตั้งชื่อเล่น (Nickname) ความยาวไม่เกิน 15 ตัวอักษรเพื่อใช้แสดงตนในการเล่นเกมก่อนได้รับสิทธิ์เข้าห้อง
4. [ ] ระบบต้องรองรับจำนวนผู้เล่นเชื่อมต่อเข้าห้องเดียวกันได้อย่างน้อย 200 ยูสเซอร์พร้อมกันโดยไม่มีปัญหาคอขวด
5. [ ] ป้องกันปัญหารุ่นชื่อซ้ำในห้องเดียวกันโดยคืนคำแจ้งเตือนว่า "ชื่อเล่นนี้ถูกใช้งานแล้วในห้องนี้"

---

## 🛠 Technical Tasks (งานพัฒนาที่ต้องทำ)
- [ ] สร้างหน้า UI เพจสำหรับผู้ใช้ Join หน้าแรก (`/join` หรือ `/game/join`)
- [ ] พัฒนาแบบฟอร์มตรวจสอบสิทธิ์และป้อนค่าฟิลด์ชื่อเล่นและรหัสห้อง (Validation rules สำหรับความยาวตัวอักษร)
- [ ] พัฒนาช่องทางการดึง Connection ID และทำการผูกยึด Socket เข้ากับห้องนั้นๆ บนฝั่ง WebSocket Server
- [ ] ตรวจเช็ครายชื่อยูสเซอร์ปัจจุบันในห้องบนหน่วยความจำเพื่อป้องกันชื่อเล่นชนกัน (Duplicate checking)
