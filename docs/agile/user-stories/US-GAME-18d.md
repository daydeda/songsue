# User Story: US-GAME-18d - Quiz — Staff ออกโจทย์ ผู้เข้าร่วมตอบแข่งความเร็ว

**Status:** 🏗 Planned (วางแผน)
**Epic:** [Multi-Interactive Game Session (WebRTC/WebSocket)](../01-product-backlog.md#15-multi-interactive-game-session-webrtc--เกมร่วมกันแบบ-real-time-ในงาน)
**Owner:** Developer
**Version:** 1.0 | **Last Updated:** 2026-06-18

---

## 📖 Description
**ในฐานะ** Staff (Host)  
**ฉันต้องการ** ส่งคำถามควิซ (Quiz) ให้เด้งขึ้นบนอุปกรณ์มือถือของผู้เข้าร่วมทุกคนเพื่อแข่งกันเลือกข้อที่ถูกต้องและส่งผลเร็วที่สุด  
**เพื่อให้** สร้างบรรยากาศสนุกสนาน มีการเรียนรู้ร่วมกัน และกระตุ้นความกระตือรือร้นในการตอบคำถามอย่างเป็นธรรม  

---

## ✅ Acceptance Criteria
1. [ ] คำถามและตัวเลือกตอบของแต่ละข้อจะแสดงผลตรงกันบนจอภาพโทรศัพท์ของผู้ร่วมเล่นทุกคนทันทีที่ Host กดสลับแสดงโจทย์ข้อนั้นๆ
2. [ ] มีเวลานับถอยหลัง (Countdown Timer) เช่น 10, 20 หรือ 30 วินาที แสดงบนหน้าจอคู่ขนานทั้งฝั่งผู้เล่นและจอ Host
3. [ ] ระบบคะแนนต้องเป็นแบบอัตราความเร่ง: หากตอบถูกและตอบเร็ว จะได้คะแนนในสัดส่วนที่สูงกว่าคนที่ตอบช้ากว่า
4. [ ] บันทึกและอ้างอิงเวลาที่ตอบจากเวลาที่ข้อความคำตอบส่งมาถึงเซิร์ฟเวอร์ (Server-side received timestamp) เท่านั้น เพื่อป้องกันไม่ให้ผู้ใช้แฮกเวลารับส่งในเครื่อง
5. [ ] เมื่อเวลานับถอยหลังหมดลง ระบบจะสรุปจำนวนเปอร์เซ็นต์ของผู้เล่นที่เลือกตัวเลือกต่างๆ ในรูปแบบของแผนภูมิแท่ง (Bar Chart) อัปโหลดสดบนจอฉายภาพหลัก

---

## 🛠 Technical Tasks (งานพัฒนาที่ต้องทำ)
- [ ] ออกแบบโมเดลคำถาม-คำตอบควิซและระบบตรวจคะแนนบนฐานข้อมูลและสคีมาบอร์ดเกมส์
- [ ] พัฒนาฟังก์ชันคำนวณคะแนนตามสัดส่วนเวลา:
  ```typescript
  const score = isCorrect ? Math.round(basePoints * (1 + (timeLeft / totalTime))) : 0;
  ```
- [ ] ตั้งค่าการ Broadcast สัญญาณเริ่มควิซ (`quizStart`) และยุติรับผลควิซเมื่อหมดเวลา (`quizTimeOut`)
- [ ] พัฒนา component แผนภูมิแท่งสรุปสถิติตัวเลือกบนหน้าจอโฮสต์โดยใช้ CSS หรือ SVG ในการเรนเดอร์
