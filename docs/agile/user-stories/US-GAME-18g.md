# User Story: US-GAME-18g - Leaderboard แสดงอันดับหลังแต่ละรอบ

**Status:** 🏗 Planned (วางแผน)
**Epic:** [Multi-Interactive Game Session (WebRTC/WebSocket)](../01-product-backlog.md#15-multi-interactive-game-session-webrtc--เกมร่วมกันแบบ-real-time-ในงาน)
**Owner:** Developer
**Version:** 1.0 | **Last Updated:** 2026-06-18

---

## 📖 Description
**ในฐานะ** ผู้เข้าร่วมงาน (Student/Guest)  
**ฉันต้องการ** เห็นตารางจัดอันดับผู้นำ (Leaderboard) ของตนเองและผู้เล่นอื่นอัปเดตสถานะความคืบหน้าทันทีหลังจบคำถามหรือจบรอบกิจกรรมแต่ละช่วง  
**เพื่อให้** สามารถติดตามสถานการณ์ความใกล้เคียงของคะแนนและอันดับของตนเองเทียบกับผู้นำ สร้างความท้าทายในการแข่งขันอย่างต่อเนื่อง  

---

## ✅ Acceptance Criteria
1. [ ] หน้าจอฉายหลักของ Host แสดงผลบอร์ดผู้นำ Top 5 คนแรก พร้อมชื่อเล่นและผลรวมคะแนนสะสมข้ามรอบ
2. [ ] มีเอฟเฟกต์การเลื่อนขยับสลับตำแหน่งของยูสเซอร์ขึ้นลง (Position switching animations) เมื่อผลคะแนนของรอบใหม่มาคำนวณเบียดตำแหน่งเดิม
3. [ ] หน้าจอโทรศัพท์มือถือของผู้เล่นต้องอัปเดตแจ้งผลลัพธ์ว่าตนเองอยู่ในอันดับที่เท่าไรของห้อง และระบุส่วนต่างคะแนนที่ต้องการเพื่อไต่อันดับขึ้นไป
4. [ ] รองรับการประมวลผลกรณีที่มีผู้เล่นหลายคนทำคะแนนรวมเท่ากัน โดยยังคงสิทธิ์การเรียงอันดับอย่างยุติธรรมตามเงื่อนไข (เช่น ความไวในการตอบรวม หรือการสุ่มอันดับที่เป็นระบบ)

---

## 🛠 Technical Tasks (งานพัฒนาที่ต้องทำ)
- [ ] พัฒนาโมดูลจัดอันดับคะแนนและเก็บคะแนนรวมสะสม (Scoreboard Aggregation Engine) บนหน่วยความจำชั่วคราว
- [ ] นำเฟรมเวิร์กจัดการ Animation บน React (เช่น `framer-motion` หรือ CSS keyframes) มาใช้ตกแต่งการขยับสไลด์ในหน้าจอผลบอร์ดผู้นำ
- [ ] พัฒนา API หรือ WebSocket message ส่งผลส่วนต่างอันดับและคะแนนของรายบุคคลเพื่อนำไปแสดงผลตรงบนอุปกรณ์ส่วนตัวของผู้ใช้แต่ละคน
