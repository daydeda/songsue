# User Story: US-GAME-18h - เชื่อมคะแนนเกมกับระบบ House Points

**Status:** 🏗 Planned (วางแผน)
**Epic:** [Multi-Interactive Game Session (WebRTC/WebSocket)](../01-product-backlog.md#15-multi-interactive-game-session-webrtc--เกมร่วมกันแบบ-real-time-ในงาน)
**Owner:** Developer
**Version:** 1.0 | **Last Updated:** 2026-06-18

---

## 📖 Description
**ในฐานะ** Staff (Admin)  
**ฉันต้องการ** แปลงคะแนนดิบสะสมที่ผู้เข้าร่วมที่ใช้บัญชี CMU ได้รับจากการเล่นเกม มาคำนวณและเพิ่มเข้าไปเป็นคะแนนบ้านดิจิทัล (House Points) ของคณะให้อัตโนมัติ  
**เพื่อให้** เกมการร่วมสนุกส่งผลและทวีความสำคัญต่อระบบการสะสมคะแนนบ้านของบ้านสีต่างๆ กระตุ้นความร่วมมือของสมาชิกประจำบ้านแต่ละหลัง  

---

## ✅ Acceptance Criteria
1. [ ] สิทธิ์การรับโอนคะแนนจากเกมไปเป็น House Points จำกัดเฉพาะผู้เล่นที่ล็อกอินหรือผูกบัญชีตรวจสอบด้วยอีเมลลงท้าย `@cmu.ac.th` เท่านั้น
2. [ ] Admin สามารถตั้งค่าเปิด-ปิดฟังก์ชันการแลกคะแนน (Award Points: ON/OFF) และกำหนดอัตราตัวคูณการแปลงแต้ม (เช่น 10 คะแนนเกม = 1 คะแนนบ้าน) ได้ผ่าน Host Control Panel ก่อนเริ่มเกม
3. [ ] ทุกขั้นตอนการแจกคะแนนโดยอัตโนมัติจะต้องถูกเขียนประวัติการดำเนินกิจกรรม (Log entry) ลงสู่ตารางความปลอดภัย `auditLogs` เหมือนกับการบันทึกคะแนนปกติ โดยระบุชื่อเซสชันเกมและคีย์ห้องอ้างอิงชัดเจน
4. [ ] ระบบต้องสะท้อนการเปลี่ยนแปลงคะแนนที่แอดเพิ่มเข้าไปในตารางบอร์ดบ้านหลักโดยทันที พร้อมแจ้งเตือนผลสำเร็จให้ผู้ใช้ทราบผ่านทางโทรศัพท์

---

## 🛠 Technical Tasks (งานพัฒนาที่ต้องทำ)
- [ ] พัฒนาระบบเชื่อมโยงโปรไฟล์ผู้ใช้ (Account mapping) ระหว่างชื่อเล่นชั่วคราวในห้องเกมส์และบัญชีนักศึกษาจริงในฐานข้อมูลหลัก
- [ ] ออกแบบ API หรือคำสั่งบริการแจกคะแนนอัตโนมัติพร้อมเงื่อนไขการตรวจสอบความปลอดภัย ป้องกันช่องโหว่การโจมตีเพื่อปั๊มแต้มบ้าน (Rate limiting / verification)
- [ ] แทรกฟังก์ชันการประมวลผลธุรกรรมฐานข้อมูล (SQL Transactions) ร่วมกับระบบความปลอดภัย Hash Chain ตาราง `auditLogs`
- [ ] ทดสอบประสิทธิภาพการทำงานและระยะเวลาตอบสนองการเขียนข้อมูลตารางพร้อมกันจำนวนหลายแถว
