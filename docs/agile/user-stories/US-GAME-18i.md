# User Story: US-GAME-18i - Host จัดการ Session และปิดห้องอย่างสง่างาม

**Status:** 🏗 Planned (วางแผน)
**Epic:** [Multi-Interactive Game Session (WebRTC/WebSocket)](../01-product-backlog.md#15-multi-interactive-game-session-webrtc--เกมร่วมกันแบบ-real-time-ในงาน)
**Owner:** Developer
**Version:** 1.0 | **Last Updated:** 2026-06-18

---

## 📖 Description
**ในฐานะ** Staff (Host)  
**ฉันต้องการ** ควบคุมการดำเนินงาน ยุติกิจกรรม หรือกดสั่งปิดเซสชันการเล่นเกม (Close Room) ได้ทุกเมื่อตลอดระยะเวลาการเล่น  
**เพื่อให้** สามารถควบคุมสถานการณ์การจัดกิจกรรมหน้างานกรณีเกิดเหตุฉุกเฉินได้ และทำการปิดห้องเคลียร์ทรัพยากรระบบออกได้อย่างถูกต้องสมบูรณ์  

---

## ✅ Acceptance Criteria
1. [ ] Host มีปุ่มลัดคำสั่ง "ปิดห้องเกมส์" (Close/Terminate Room) ที่โดดเด่นบนแผงควบคุมระบบ
2. [ ] เมื่อกดปิดห้อง ระบบจะทำการ Broadcast ข้อความ "ขอบคุณทุกคนที่ร่วมสนุกกิจกรรม..." และนำทางผู้เล่นที่เหลือทั้งหมดออกจากเซสชันกลับหน้าหลักโดยอัตโนมัติ
3. [ ] กรณีตรวจพบสัญญาณ Host หลุดออกจากการเชื่อมต่อเป็นเวลานานเกินกว่า 60 วินาที ระบบจะพ่นข้อความเตือนให้ผู้เข้าร่วมรับทราบว่า "ผู้จัดขาดการเชื่อมต่อชั่วคราว..."
4. [ ] ข้อมูลประวัติผลควิซ และผลคะแนนโหวตโพลดิบในห้องที่เพิ่งปิด จะถูกจัดเก็บสำรองลงฐานข้อมูลถาวรฝั่งเซิร์ฟเวอร์ และเปิดให้แอดมินเข้าไปดาวน์โหลดไฟล์ข้อมูลย้อนหลังได้ใน 24 ชั่วโมงก่อนเคลียร์ทิ้ง
5. [ ] คืนหน่วยความจำชั่วคราว (Clear memory/Redis cache) ของตัวแปรห้องนั้นๆ ออกทันทีเมื่อห้องปิดสมบูรณ์

---

## 🛠 Technical Tasks (งานพัฒนาที่ต้องทำ)
- [ ] พัฒนา WebSocket handler ตรวจจับกรณี Host ตัดการเชื่อมต่อ (`disconnect` event) และกลไกหน่วงเวลา 60 วินาที (Grace period timer) ก่อนส่งสัญญาณตัดระบบผู้ใช้อื่น
- [ ] ออกแบบ UI หน้าจอสิ้นสุดกิจกรรม (Graceful end screen) สำหรับฝั่งผู้ร่วมเล่นและหน้าจอฉายภาพหลัก
- [ ] เขียนสคริปต์ย้ายสเตตข้อมูลเกมและโพลจาก In-memory ไปจัดเก็บลงฐานข้อมูลชั่วคราว พร้อมระบบตั้งค่าลบไฟล์ตามตารางเวลา (TTL/Cron clean-up)
- [ ] ทดสอบกระบวนการปิดห้องและตรวจสอบการทำงานของ Garbage Collection ว่าหน่วยความจำลดลงกลับสู่ระดับปกติจริง
