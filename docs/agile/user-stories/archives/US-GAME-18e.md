# User Story: US-GAME-18e - Mini-game — Reaction / กดปุ่มแข่งความเร็ว

**Status:** 🏗 Planned (วางแผน)
**Epic:** [Multi-Interactive Game Session (WebRTC/WebSocket)](../01-product-backlog.md#15-multi-interactive-game-session-webrtc--เกมร่วมกันแบบ-real-time-ในงาน)
**Owner:** Developer
**Version:** 1.0 | **Last Updated:** 2026-06-18

---

## 📖 Description
**ในฐานะ** ผู้เข้าร่วมงาน (Student/Guest)  
**ฉันต้องการ** เล่นมินิเกมแข่งความเร็วในการกดปุ่มทันทีเมื่อสัญญาณเตือนเปลี่ยนจากสีแดงเป็นสีเขียว  
**เพื่อให้** แข่งขันความเร็วในการตอบสนอง (Reaction Time) ของตนเองกับผู้เข้าร่วมคนอื่น และสอยตำแหน่งผู้ชนะที่มีความไวประสาทตาและมือดีที่สุดในกิจกรรม  

---

## ✅ Acceptance Criteria
1. [ ] หน้าจอโทรศัพท์มือถือของผู้เล่นต้องขึ้นหน้าปุ่มขนาดใหญ่สีแดงที่ระบุว่า "กรุณารอสัญญาน..."
2. [ ] สัญญาณเริ่มสปินสุ่มเปลี่ยนเป็นสีเขียวเขียนว่า "GO!" (หรือ "กดทันที!") จะถูกยิงไปที่ทุกคนพร้อมกันหลัง Host กดสตาร์ทการทดสอบ
3. [ ] ผู้ใช้ที่กดปุ่มก่อนที่คำสั่งจะเปลี่ยนเป็นสีเขียว (กดก่อนสัญญาน) จะถูกตัดสินว่าทำผิดกติกา (Foul) และได้คะแนนเป็น 0 ในรอบนั้น
4. [ ] การจับคู่เวลาความเร็วในการตอบสนองเป็นระดับมิลลิวินาที (ms) โดยนำเวลาส่งสัญญาณจริงมาหักลบกับเวลาตอบรับที่เซิร์ฟเวอร์ได้รับ
5. [ ] หน้าจอของโฮสต์จะต้องสรุปรายชื่อ Top 10 ผู้เล่นที่ทำเวลาตอบสนองเร็วที่สุดหลังจบรอบการทดสอบ

---

## 🛠 Technical Tasks (งานพัฒนาที่ต้องทำ)
- [ ] เขียนฟังก์ชันสุ่มวินาทีการเปลี่ยนสถานะของปุ่มเป็นสีเขียวบน Server (ระหว่าง 2-5 วินาที) แล้วพ่น Event ไปที่ไคลเอนต์
- [ ] พัฒนาฟังก์ชันคำนวณและตรวจสอบเงื่อนไขเวลาความไวการกด และคัดแยกคนที่กดยึดจังหวะฟาวล์ (Foul tap checking)
- [ ] ทดสอบความแม่นยำในการจับเวลามิลลิวินาทีของ WebSocket ภายใต้เครือข่าย WiFi 5GHz และเครือข่ายมือถือทั่วไป
- [ ] ออกแบบเอฟเฟกต์สีหน้าต่างเปลี่ยนฉับพลัน (Flash transitions) บน CSS เพื่อกระตุ้นปฏิกิริยานิ้วมือ
