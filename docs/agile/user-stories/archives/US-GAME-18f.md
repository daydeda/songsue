# User Story: US-GAME-18f - Live Poll — โหวตและเห็นผลแบบ Real-time

**Status:** 🏗 Planned (วางแผน)
**Epic:** [Multi-Interactive Game Session (WebRTC/WebSocket)](../01-product-backlog.md#15-multi-interactive-game-session-webrtc--เกมร่วมกันแบบ-real-time-ในงาน)
**Owner:** Developer
**Version:** 1.0 | **Last Updated:** 2026-06-18

---

## 📖 Description
**ในฐานะ** Staff (Host)  
**ฉันต้องการ** ตั้งโจทย์คำถามสำรวจความคิดเห็น (Poll) เพื่อให้ผู้เล่นทุกคนร่วมโหวตตัวเลือกและดูผลลัพธ์ผ่านแผนภูมิการโหวตสดที่ขยับขึ้นลงตามผลคะแนนจริง  
**เพื่อให้** เก็บผลสรุปความคิดเห็นของผู้เข้าร่วมงานในหัวข้อต่างๆ หรือทำช่วงตอบรับเพื่อมีส่วนร่วมในการตัดสินใจแบบเปิดเผยได้ในเวลาอันรวดเร็ว  

---

## ✅ Acceptance Criteria
1. [ ] ผู้ใช้เห็นรายการตัวเลือกให้เลือกทำเครื่องหมายโหวตตามรูปแบบที่ผู้จัดกำหนด (ตัวเลือกเดียว หรือ หลายตัวเลือก)
2. [ ] ผลโหวตอัปเดตแบบวินาทีต่อวินาทีบนจอฉายภาพหลัก โดยแสดงผลเป็น % และจำนวนคนที่เลือกแต่ละตัวเลือก
3. [ ] ผู้เล่นหนึ่งคน (อ้างอิง Session ID หรือชุด Client ID ของเครื่อง) มีสิทธิ์กดส่งคะแนนโหวตได้เพียงครั้งเดียวต่อหัวข้อ เพื่อป้องกันการปั๊มผลคะแนนโหวต
4. [ ] Host สามารถกดสั่ง "ปิดโหวต" (Lock Poll) ได้ตามต้องการ เพื่อหยุดรับคะแนนโหวตเพิ่ม
5. [ ] ระบบอนุญาตให้ผู้ดูแลดาวน์โหลดผลคะแนนโหวตและรายชื่อการเลือกออกมาเป็นรูปแบบไฟล์ `.csv` หลังจบกิจกรรม

---

## 🛠 Technical Tasks (งานพัฒนาที่ต้องทำ)
- [ ] ออกแบบ API โครงสร้างฐานข้อมูลสำหรับเก็บหัวข้อโหวต ตัวเลือก และบันทึกคำตอบผู้ใช้
- [ ] พัฒนาฟังก์ชันการสะสมผลการโหวตแบบเรียลไทม์ (Aggregation state) และการ Broadcast การอัปเดตค่าสัดส่วนเปอร์เซ็นต์
- [ ] พัฒนาตัวป้องกันโหวตซ้ำ (Anti-double-voting engine) โดยเปรียบเทียบจาก Token / Device ID ที่ลงทะเบียนไว้
- [ ] เขียนฟังก์ชันสำหรับแปลงผลโหวตดิบในหน่วยความจำและสร้างตัวแปลไฟล์ข้อมูลให้ออกมาเป็นรูปแบบไฟล์ CSV
