# User Story: US-OPT-19c - ย้าย checkAndAwardClosedForms ออกจาก Hot-path /api/houses

**Status:** 🏗 Planned (วางแผน)
**Epic:** [Performance & API Cache Optimization](../01-product-backlog.md#16-performance--api-cache-optimization--ความเร็วและประสิทธิภาพระบบ)
**Owner:** Developer
**Version:** 1.0 | **Last Updated:** 2026-06-18

---

## 📖 Description
**ในฐานะ** ผู้ดูแลระบบ (Admin)  
**ฉันต้องการ** ให้การเปิดดูอันดับบ้านในลีดเดอร์บอร์ดดึงผลวิเคราะห์แสดงทันทีโดยไม่ต้องไปจมกับขั้นตอนการประมวลผลคำนวณคะแนนของแบบฟอร์มประเมินที่เพิ่งปิดรับ  
**เพื่อให้** อัตราความเร็วในการโหลดคะแนนสะสมเฉลี่ยลดลง และไม่มีอาการค้างเนื่องจากการรันงานคำนวณหลังบ้านพร้อมกับการโหลดภาพ  

---

## ✅ Acceptance Criteria
1. [ ] ลบคำสั่งเรียกฟังก์ชันตรวจและให้คะแนน `checkAndAwardClosedForms()` ออกจาก API บอร์ดบ้าน `/api/houses/route.ts` อย่างสิ้นเชิง เพื่อแยกส่วนข้อมูลการจัดอันดับออกจากส่วนคำนวณเบื้องหลัง
2. [ ] โอนความรับผิดชอบในการรันระบบตรวจสอบประเมินคะแนนดังกล่าวไปไว้ที่ชุดคำสั่งงานอัตโนมัติ (Cron Job) ตามเส้นทางระบบ `/api/cron/award-points` โดยมีคาบเวลารันอัตโนมัติทุกๆ 5 นาที
3. [ ] ตัว Cron Job จะต้องทำงานอยู่ภายใต้ระบบล็อกและการจัดการสิทธิ์เพื่อความปลอดภัย โดยอนุญาตการเรียกใช้เฉพาะสัญญาณที่มาจากเซิร์ฟเวอร์เท่านั้น
4. [ ] ความเร็วเฉลี่ยในการตอบกลับหน้าอันดับบอร์ดบ้านต้องเร็วขึ้นไม่น้อยกว่า 30% เมื่อไม่มีงานเบื้องหลังค้างคา
5. [ ] มีบันทึกผลการแจกคะแนนโดยอัตโนมัติและข้อมูลเวลาที่รันเข้าสู่หน้าต่างบันทึก Log ของระบบอย่างชัดเจน

## 🛠 Technical Tasks (งานพัฒนาที่ต้องทำ)
- [ ] ค้นหาและนำโค้ดที่รันคำสั่ง `checkAndAwardClosedForms()` ออกจากไฟล์ `src/app/api/houses/route.ts` (ประมาณแถวที่ 17)
- [ ] ตรวจเช็คการตั้งค่าในไฟล์งาน `src/app/api/cron/award-points/route.ts` และยืนยันระบบเรียกใช้การประมวลผลลูปคำนวณอย่างถูกต้องเหมาะสม
- [ ] เขียนและทดสอบระบบจำลองการยิงคำสั่ง Cron แบบจำลองเพื่อตรวจดูเวลาการประมวลผล
- [ ] เช็คและทดสอบความเร็วหน้าระบบลีดเดอร์บอร์ดหลังปรับแก้โค้ด
