# Feature Spec — Month-view Calendar & External .ics Subscription (ระบบปฏิทินและการดึงข้อมูล .ics)

> **STATUS: IMPLEMENTED (shipped on PR #82, #85, #86).**

---

## 1. Context / ปัญหาและที่มา
นักศึกษาต้องการดูปฏิทินกิจกรรมรายเดือนเพื่ออำนวยความสะดวกในการวางแผนและเตรียมตัวเข้าร่วมกิจกรรมต่าง ๆ รวมถึงกิจกรรมย่อย กำหนดการสอบ หรือกำหนดการรับสมัคร นอกจากนี้ ยังต้องการช่องทางในการส่งออกรายการปฏิทินไปยังแอปพลิเคชันภายนอก เช่น Google Calendar, Apple Calendar และ Outlook เพื่อไม่ให้พลาดช่วงเวลาลงทะเบียนหรือเข้าร่วมกิจกรรมจริง

---

## 2. การออกแบบและสถาปัตยกรรม (Technical Design)

### 2.1 หน้าปฏิทินบอร์ด (Month-view Calendar UI)
* ระบบแสดงผลปฏิทินแบบรายเดือนที่หน้า `/dashboard/calendar`
* แสดงรายการแบบผสม (Hybrid) ระหว่าง:
  1. กิจกรรมเช็คอินหลัก (`events`) ที่นักศึกษามีสิทธิ์เข้าร่วม
  2. รายการบันทึกปฏิทินเสริม (`calendar_entries`) เช่น วันหยุด, สัปดาห์สอบ หรือกำหนดการย่อย
* รองรับ Rich-Text ในตัว Modal รายละเอียดปฏิทินเพื่อแสดงภาพโปสเตอร์และคำอธิบายแบบจัดรูปแบบได้อย่างสวยงาม

### 2.2 ระบบเชื่อมโยงปฏิทินภายนอก (RFC 5545 .ics Feed)
* ระบบเปิดช่องทาง API `/api/calendar/feed/[token]` เพื่อส่งออกข้อมูลไฟล์ปฏิทินมาตรฐาน `iCalendar` (.ics)
* **การข้าม Middleware:** เนื่องจากผู้ดึงข้อมูลคือ Bot/User-Agent ของผู้ให้บริการปฏิทินภายนอก (เช่น Google Calendar, iCloud) ซึ่งไม่รองรับการ Login หรือการฝัง Session Cookie ของระบบ จึงต้องตั้งค่าข้าม Proxy Middleware และอนุญาตให้เข้าถึง API นี้แบบสาธารณะ
* **ความปลอดภัยตามหลัก PDPA:**
  - เพื่อรักษาความปลอดภัยของข้อมูล กิจกรรมในปฏิทินส่วนบุคคลจึงถูกป้องกันผ่าน **Revocable Secret Token** (โทเค็นสุ่มเก็บในตาราง `calendar_feed_tokens`)
  - โทเค็นนี้ทำหน้าที่เปรียบเสมือน API Key เฉพาะบุคคลของนักศึกษาแต่ละคน
  - มีระบบตรวจสอบความถูกต้องภายใน API Route (In-Route Validation)
  - นักศึกษาสามารถกด **"Regenerate Token"** ในเมนูตั้งค่าปฏิทินได้ตลอดเวลา ระบบจะสร้างรหัสโทเค็นชุดใหม่ทับในฐานข้อมูลทันที ทำให้ลิงก์ปฏิทินเก่าใช้งานไม่ได้อีกต่อไป เพื่อความปลอดภัยสูงสุดในกรณีที่ URL รั่วไหล

## 3. ความปลอดภัยและสิทธิ์การเข้าถึง (Access Control)
* การดึงข้อมูลปฏิทินผ่าน API Feed ยังคงผ่านฟังก์ชันตรวจสอบสิทธิ์เข้าถึงเดียวกับหน้าเว็บหลัก (`src/lib/event-access.ts`)
* กรองข้อมูลอัตโนมัติตาม:
  - **บทบาทของผู้ใช้ (Allowed Roles):** คัดกรองเฉพาะบทบาทที่ได้รับอนุญาต
  - **สาขาวิชา (Allowed Majors):** คัดกรองแยกตาม ANI, DG, DII, MMIT, SE
  - **กลุ่มสัญชาติ/หลักสูตร (Thai / International):** คัดกรองตามสิทธิกลุ่มผู้เรียนภาษาไทยและต่างประเทศ
* สำหรับแอดมินหรือ Staff จะข้ามการตรวจสอบเหล่านี้ตามสิทธิ์ของบทบาท

---

## 4. โครงสร้างฐานข้อมูลย่อย (Schema Reference)
* ตาราง `calendar_entries`: เก็บรายละเอียดหัวข้อ, คำอธิบาย, วันที่จัด, และฟิลด์กรองสิทธิ์เข้าใช้งาน
* ตาราง `calendar_feed_tokens`: เก็บค่ารหัสลับ `token` และประวัติการเชื่อมต่อล่าสุด `lastUsedAt` ของนักศึกษาแต่ละคน
