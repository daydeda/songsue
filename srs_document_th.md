# เอกสารข้อกำหนดความต้องการทางซอฟต์แวร์ (Software Requirements Specification - SRS)

**ชื่อระบบงาน:** แพลตฟอร์มบริหารจัดการกิจกรรมและการจัดสรรคะแนนบ้านดิจิทัลแบบเรียลไทม์ (ActiveCAMT)  
**ฉบับที่:** 1.0 (ฉบับปรับปรุงเนื้อหาเชิงลึกระดับสารบัญรหัสโปรแกรมและการป้องกันภัยคุกคาม)  
**สถานะ:** สมบูรณ์ (พร้อมเสนอคณะกรรมการบริหารและแอดมินระบบ)  

---

## 1. บทนำ (Introduction)

### 1.1 วัตถุประสงค์ (Purpose)
เอกสารฉบับนี้กำหนดรายละเอียดทางเทคนิค ข้อกำหนดการทำงาน (Functional Requirements) และข้อกำหนดที่เกี่ยวข้องกับสถาปัตยกรรม (Non-Functional Requirements) ของโครงการ **ActiveCAMT** เพื่อยกระดับความโปร่งใส ปลอดภัย และมีประสิทธิภาพสูงสุดในการดำเนินงานกิจกรรมเชิงสร้างสรรค์ของมหาวิทยาลัย

### 1.2 ขอบเขตของระบบ (System Scope)
ระบบทำงานในรูปแบบของสถาปัตยกรรมโมดูลาร์มอนอลิธ (Modular Monolith) ที่รันบน Next.js 16 (App Router) ประกอบด้วยระบบจัดการผู้ใช้งาน แผงควบคุมผู้ควบคุมงาน ระบบลงทะเบียนด้วย Dynamic Secure QR Code ระบบสแกนเนอร์ความเร็วสูง ระบบแบบสอบถามไดนามิกสไตล์ Google Forms ระบบแสดงผลสะสมคะแนนแบบเกมมิ่งบอร์ด และระบบบันทึกความปลอดภัยข้อมูลประวัติการรักษาพยาบาล (Immutable Audit Trails)

## 3. สารบัญคำอธิบายไฟล์และการทำงานเชิงลึกอย่างละเอียด (Comprehensive File Directory)

### 3.1 กลุ่มไฟล์ระบบ ความปลอดภัย และการตั้งค่ากลาง (Configuration & Auth)

#### 📂 [src/auth.ts](file:///E:/OnlyWork/SMO Meetings/Web/activecamt/src/auth.ts)
* **หน้าที่การทำงานหลัก:** ควบคุมฟังก์ชันล็อกอิน ยืนยันตัวตน และจัดการความปลอดภัยเซสชันผ่าน NextAuth v5 (Beta) ร่วมกับระบบ Google Provider และ `@auth/drizzle-adapter`

  * บันทึกข้อมูลและสิทธิ์ `role` (student, prof, officer, admin) ลงไปใน JWT token
  * **[CRITICAL LOGIC] ระบบซิงค์เซสชันไดนามิก:** ภายในฟังก์ชัน `jwt` callback มีการสร้างเงื่อนไขดักจับสัญญาน `trigger === "update"` เพื่อเขียนเซสชันคุกกี้บนบราวเซอร์นักศึกษาใหม่ทันทีเมื่อทำ Onboarding หรือแก้ไขโปรไฟล์สำเร็จ ทำให้นักศึกษาสามารถเข้าถึงแดชบอร์ดได้ทันทีโดยไม่ต้อง Log out เพื่อดึงข้อมูลสิทธิ์ใหม่
  * นิยามออปชันตรวจสอบสิทธิ์และค่าล้าสมัยของเวลาเซสชัน (Session Expiry)

#### 📂 [src/db/schema.ts](file:///E:/OnlyWork/SMO Meetings/Web/activecamt/src/db/schema.ts)
* **หน้าที่การทำงานหลัก:** นิยามโครงสร้างโมเดลความสัมพันธ์ของตารางฐานข้อมูล (Relational Database Design) บนภาษา TypeScript ผ่าน Drizzle ORM
* **รายละเอียดโลจิกภายใน:**
  * นิยามตาราง `users` เก็บสิทธิ์ รูปโปรไฟล์ ข้อมูลสิทธิ์ และสถานะโปรไฟล์ (`profileCompleted`)

  * นิยามตาราง `auditLogs` สำหรับบันทึกความปลอดภัย ทุกการกดดูประวัติสุขภาพนักศึกษาของเจ้าหน้าที่ (ตารางนี้เป็นแบบเพิ่มต่อท้ายเท่านั้น ปราศจากการเรียกใช้งานคำสั่ง UPDATE หรือ DELETE)
  * นิยามตาราง `events` (รายละเอียดการจอง โควตา และแบบฟอร์มประเมินในตัว), `attendance` (บันทึกเช็คอิน), `houses` (คะแนนบ้าน) และ `points` (ประวัติการปรับคะแนนและโบนัสบ้าน)

* **หน้าที่การทำงานหลัก:** สถาปนาการเชื่อมต่อโครงสร้างไปยังฐานข้อมูลหลัก PostgreSQL
* **รายละเอียดโลจิกภายใน:**
  * เรียกอ่านค่า `DATABASE_URL` จากสภาพแวดล้อมระบบ (.env)
  * ใช้งานไลบรารีขับเคลื่อนฐานข้อมูล `postgres` ร่วมกับ Drizzle ORM
  * **[PERFORMANCE] Connection Pooler:** ควบคุมการเชื่อมต่อแบบ Singleton เพื่อป้องกันสภาวะเชื่อมต่อเต็ม (Max Connection Limit Exceeded) ในระหว่างที่นักศึกษาเรียกใช้แอปพร้อมกันหลายคน

### 3.3 กลุ่มไฟล์ระดับหลังบ้านและการจัดการสิทธิ์ (Backend API Route Handlers)

#### 📂 [src/app/api/upload/route.ts](file:///E:/OnlyWork/SMO Meetings/Web/activecamt/src/app/api/upload/route.ts)
* **หน้าที่การทำงานหลัก:** บริการตรวจสอบความปลอดภัยและจัดเก็บรูปภาพโปสเตอร์กิจกรรมและรูปโปรไฟล์นักศึกษาลงสู่ดิสก์ `/public/uploads/`
* **รายละเอียดโลจิกภายใน:**
  * ตรวจสอบสิทธิ์เซสชัน NextAuth ป้องกันบอทหรือสแปมขอนอกระบบเข้าใช้งาน
  * ตรวจกรองความปลอดภัยของประเภทไฟล์ (MIME Type) และตรวจสอบสกุลไฟล์ (`.jpg`, `.png`, `.webp`, etc.) เพื่อป้องกัน Stored XSS ในระดับไฟล์ระบบ
  * **[NEW - SERVER-SIDE LIMIT] สกัดกั้นไฟล์ใหญ่เกิน 5MB:**
    ```typescript
    const MAX_SIZE = 5 * 1024 * 1024;
    if (file.size > MAX_SIZE) {
      return NextResponse.json({ error: "File size exceeds the 5MB limit." }, { status: 400 });
    }
    ```
    หากเกิดความพยายามเจาะทะลุกรอบหน้าบ้านเข้ามาด้วยไฟล์ดิบขนาดใหญ่ ระบบหลังบ้านจะสั่งทำลายทรานแซกชันการเขียนไฟล์ และตอบกลับรหัสความผิดพลาด 400 Bad Request ทันที

#### 📂 [src/app/api/realtime/route.ts](file:///E:/OnlyWork/SMO Meetings/Web/activecamt/src/app/api/realtime/route.ts)
* **หน้าที่การทำงานหลัก:** คอนโทรลเลอร์จัดทำ Persistent Server-Sent Events (SSE) คลื่นสตรีมสัญญาณการเปลี่ยนแปลงเพื่อส่งข้อมูลสู่เบราว์เซอร์ผู้ใช้งาน
* **รายละเอียดโลจิกภายใน:**
  * ส่งข้อมูลแบบสตรีมมิ่งที่ใช้ท่อสัญญาณ Keep-alive และส่ง Heartbeat pings ทุก ๆ 15 วินาทีเพื่อป้องกันระบบพร็อกซี่ตัดช่องการสื่อสาร
  * **[CRITICAL SECURITY] สิทธิคุ้มครองส่วนบุคคลแบบแยกพิกัดข้อมูล (PDPA Filter):**
    * หากบราวเซอร์ระบุตัวตนเป็นสิทธิ์ **Admin** สัญญาณจะส่งข้อมูลกิจกรรมทั้งหมด (ชื่อผู้เช็คอิน, ประวัติแอดมิน, รายการคะแนน)
    * หากเป็นสิทธิ์ **Student** ตัวกรองความปลอดภัยจะสกัดข้อมูลชื่อผู้เช็คอินของนักศึกษาคนอื่นและข้อมูลเชิงลึกทิ้งทั้งหมด โดยจะปล่อยผ่านเฉพาะผลคะแนนคะแนนสะสมของบ้านและสัญญาณการแจ้งเตือนสร้างกิจกรรมใหม่เท่านั้น เพื่อความโปร่งใสและปฏิบัติตามมาตรฐาน PDPA อย่างเคร่งครัด

#### 📂 [src/lib/realtime-emitter.ts](file:///E:/OnlyWork/SMO Meetings/Web/activecamt/src/lib/realtime-emitter.ts)
* **หน้าที่การทำงานหลัก:** ตัวเชื่อมส่งข้อความและสัญญาณข้ามโปรเซส (Cross-Process IPC Broker) เพื่อเอาชนะการแยกหน่วยความจำของ Next.js Worker Threads
* **รายละเอียดโลจิกภายใน:**
  * Next.js ทำงานภายใต้การแยกโปรเซส (Isolation) ซึ่งส่งผลให้ Memory-based Event Emitters ทั่วไปข้ามเขตการประมวลผลไม่ได้
  * ไฟล์นี้แก้ปัญหาโดยเขียนข้อความสั้น ๆ ลงดิสก์ `/scratch/realtime-events/` และใช้ `fs.watch` คอยสแกนระบบไฟล์ เมื่อพบไฟล์ขยะหรือไฟล์พิกัดสัญญาณจะส่งต่อเข้าสู่กระแส SSE ทันที แล้วทำการลบไฟล์ทิ้งอัตโนมัติ (Unlink) รักษาเสถียรภาพพื้นที่เก็บข้อมูล
