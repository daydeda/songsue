## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **ร้านค้า merch ใหม่** — สั่งของได้จากในระบบเลย เลือกไซซ์/แบบ (variant) ได้ โอนเงินผ่าน PromptPay/QR ของร้าน แล้วอัปสลิปโอนเงินเข้าไป รอแอดมินกดอนุมัติ
- **ประกาศสำคัญหน้า Dashboard อัปเดตได้สด** — แบนเนอร์ประกาศไม่ฝังตายในโค้ดแล้ว แอดมินแก้ข้อความ/เปิด-ปิดได้เอง มีตัวหนา ลิงก์ สี และพรีวิวสด ไม่ต้อง deploy ใหม่
- **โลโก้ประจำบ้าน + ชื่อบ้านภาษาไทย** — เพิ่มโลโก้มาสคอตทั้ง 4 บ้าน (มอม/โต/ลวง/มกร) โชว์บนโพเดียม leaderboard, การ์ดสถิติบ้านหน้า Dashboard และบัตรนักศึกษา (Digital ID) + เปลี่ยนชื่อบ้านเป็น บ้านมอม / บ้านโต / บ้านลวง / บ้านมกร
- **การ์ดกิจกรรมหน้า Dashboard** — แก้ป้ายวันที่ / สถานะ LIVE / คะแนน ที่เคยโดนรูปโปสเตอร์บัง ให้ลอยขึ้นมาเห็นชัดตามเดิม

## ฝั่งทีม (technical changelog)

### ร้านค้า merch (ของใหม่)
- ตาราง DB ครบชุด: `shop_settings`, `shop_products`, `shop_variants`, `shop_orders`, `shop_order_items` — line item snapshot ชื่อ/ราคา/variant ไว้ ลบสินค้าทีหลังก็ไม่กระทบประวัติออเดอร์
- สินค้ามี variant แยกไซซ์ สต็อกแยกต่อ variant มีช่วงเวลาลดราคา (sale window) และคำอธิบายแบบ rich text
- หน้าร้านฝั่งนักศึกษา + หน้าจัดการร้านฝั่งแอดมิน พร้อม API ครบ (สินค้า/ออเดอร์/settings)
- flow อนุมัติออเดอร์: ผู้ซื้อสั่ง → โอนเงิน → อัปสลิป → แอดมินรีวิว approve/reject
- สลิปเก็บใน private bucket เสิร์ฟผ่าน endpoint ที่ auth-guard เท่านั้น
- shared `RichTextEditor` + `stripRichText` + image upload helper, ใส่ nav link + i18n (en/th/my/zh), gate ร้านไว้ที่ super_admin/admin

### ประกาศหน้า Dashboard
- ย้ายแบนเนอร์ประกาศจาก JSX ฝังตาย → singleton ใน DB ที่ super_admin/admin แก้ได้ (body + เปิด/ปิด) ไม่ต้อง redeploy
- migration แบบ idempotent seed ข้อความเดิมไว้ deploy ครั้งแรกหน้าตาไม่เปลี่ยน
- `GET /api/announcement` (public) สำหรับแบนเนอร์ + `GET/PUT /api/admin/announcement` gate ด้วย roles array, validate ด้วย Zod, audit-log ใน transaction
- editor เต็มความกว้าง + toolbar (Bold / Link / Color) แบบ toggle ได้ พร้อมพรีวิวสดที่ตรงกับแบนเนอร์จริง
- render ประกาศผ่าน `parseRichText` ตัวเดียวกับคำอธิบายกิจกรรม → **ตัวหนา**, `[text](url)`, `{{color:#HEX|text}}` แสดงผลจริง ไม่ขึ้นเป็น markup ดิบ
- i18n ครบ 4 ภาษา (EN/TH/MY/CN)

### บ้าน (Houses)
- เพิ่มโลโก้มาสคอต 4 บ้าน (Mom/To/Luang/Makon) ลบพื้นหลัง + ย่อให้เบาสำหรับมือถือ (~130KB/รูป) ถ้าบ้านไหนไม่มี mapping จะ fallback เป็นไอคอน Trophy
- ชื่อบ้านภาษาไทยอ่านว่า บ้านมอม / บ้านโต / บ้านลวง / บ้านมกร — prefix "บ้าน" ฝังใน i18n key, จุดที่ต่อ "{name} House" guard ไว้ไม่ให้ไทยซ้ำคำ, ตัด prefix ซ้ำใน audit-reason translator

### Security & Performance (quick wins)
- ซ่อน `qrToken` (credential เช็คอินที่ replay ได้) ออกจาก route รายละเอียดนักศึกษาฝั่งแอดมิน
- sanitize ค่า `{{color:...}}` ให้อยู่ใน allowlist ปิดช่อง stored-XSS แบบ attribute breakout (เช่น `red" onmouseover="...`) — ค่า hex/ชื่อสีที่ถูกต้องไม่กระทบ
- guard สคริปต์ CLI ที่อันตราย/มีสิทธิ์สูง (`db:reset`, `db:seed`, `promote-admin`, `elevate-admin`) — ไม่ยอมรันใส่ DB remote/prod ถ้าไม่ใส่ `CONFIRM=yes`
- docker-compose: เอา public port 5432 ออก (DB เป็น internal-only) และบังคับตั้ง `POSTGRES_PASSWORD` (เลิกใช้ default อ่อน ๆ)
- เพิ่ม index `idx_score_history_timestamp` รองรับ `ORDER BY timestamp DESC` ของ leaderboard/dashboard

### Shop — แก้ตามหลัง
- rich text: markup ตัวหนา/สี/ลิงก์ ครอบข้ามบรรทัดได้แล้ว (ใช้ `[\s\S]` แทน flag `s` ให้เข้ากับ tsconfig target ก่อน es2018 — flag เดิมจะทำให้ `next build` พัง) แก้ข้อความ "How to pay" ที่เคยขึ้นเป็น markup ดิบ
- ขยายเพดานชื่อ variant 60 → 200 (คอลัมน์เป็น text ไม่จำกัด ลิมิตเดิมแคบเกิน)
- ชื่อ variant ยาว ๆ wrap แทนล้นจอ — chip เลือก variant, สรุปออเดอร์, ประวัติออเดอร์, แถวสินค้า/ออเดอร์ฝั่งแอดมิน รองรับชื่อยาวทุกความกว้าง
