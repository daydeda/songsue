# ActiveCAMT — อัปเดต 14–16 มิ.ย. 69

ช่วง: 14–16 มิ.ย. 69 · สรุปไว้สำหรับลง Discord และให้ทีมที่เกี่ยวข้องอ่าน

---

## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **แท็บประวัติกิจกรรม (Event History) เด่นขึ้น** — ดันขึ้นมาบน nav หาง่ายขึ้น พร้อม breakpoint ที่เป็นมิตรกับ iPad/แท็บเล็ตมากขึ้น
- **รายชื่อสมาชิกบ้าน (House Rosters)** — ดูได้ว่าใครอยู่บ้านไหน + แอดมิน export รายชื่อผู้เข้าร่วมกิจกรรมออกมาได้
- **ยกเลิกลงทะเบียนปลอดภัยขึ้น** — ยกเลิกการลงทะเบียนไม่ได้แล้วหลังเลยเวลาปิดรับ และมีหน้าต่างยืนยันก่อนถอนชื่อกันกดพลาด
- **SMO เข้าหน้าสแกนได้เอง** — บทบาท SMO เห็นปุ่ม Admin Panel และกดเข้าไปเจอหน้าสแกน QR เช็คอินได้โดยตรง (สแกนอย่างเดียว ไม่เห็นส่วนอื่นของแอดมิน)
- **ฟอร์มประเมินเลื่อนได้ลื่นบนมือถือ** — ฟอร์มประเมินที่มีดาวให้คะแนนเลื่อนได้เต็มจอ หัวฟอร์มกระชับลงบนมือถือ และรีเซ็ตการเลื่อนทุกครั้งที่ขึ้นหน้าใหม่
- **หน้าจัดการผู้ใช้ (Manage User) ใช้บนมือถือได้จริง** — โมดัลแก้ข้อมูลผู้ใช้ปรับเป็นเลื่อนได้ ไม่ล้นจอ/ไม่โดนตัด และเพิ่มช่องแก้ **คำนำหน้า (prefix)** ได้แล้ว

## ฝั่งทีม (technical changelog)

### ความเป็นส่วนตัวข้อมูลการแพทย์ (PDPA)
- ฝ่าย registration เห็นแค่ **สัญญาณ** ว่ามีภาวะทางการแพทย์ (เป็นหมวดแบบ bullet ที่แปลแล้ว) แต่ไม่เห็น **รายละเอียด** — รายละเอียด + meds-check detail เปิดให้เฉพาะแอดมิน
- ปิดช่องรั่วที่ badge meds-check เคยบอกใบ้ว่าใครมีภาวะทางการแพทย์
- emergency contact ยังเห็นได้ตามเดิมสำหรับ role ที่ดูแลงาน

### สิทธิ์เข้าถึงของ SMO (scanner-only)
- ให้ role SMO เข้า `/admin` ได้เฉพาะหน้าสแกน — ตัดสินใจ admin landing ฝั่ง server เพื่อให้ SMO เด้งไปหน้าสแกนตรง ๆ
- โชว์ปุ่ม Admin Panel ให้ SMO แล้ว route ไปหน้าสแกน
- เปิดทางที่ `src/proxy.ts` (middleware) ให้ SMO ผ่านเข้า `/admin` แบบ scanner-only — จุดนี้รันก่อน gate อื่นและพลาดง่าย
- รวม policy "SMO สแกนได้อย่างเดียว" ให้เป็น single source เดียวข้าม gating layer ทั้งหมด (เลิกกระจายตรรกะหลายที่)
- กันบั๊ก lockout: user ที่ `roles[]` ว่างแต่มี `role` เดี่ยวจะไม่โดนล็อกออกจากแอดมิน

### บ้าน (Houses) & activity feed
- ไม่ผูกกิจกรรมที่ไม่มีคนเข้าร่วม / ให้ 0 คะแนน เข้ากับบ้านใด ๆ แล้ว
- กิจกรรมที่ไม่มีการให้คะแนนยังคงอยู่ในฟีด แต่ระบุว่าไม่สังกัดบ้าน (no house)
- เพิ่มรายชื่อสมาชิกบ้าน + export รายชื่อผู้เข้าร่วมกิจกรรมฝั่งแอดมิน
- เพิ่มสคริปต์ diagnostic + recovery สำหรับ activity feed ไว้ตรวจ/กู้ข้อมูล

### Navigation
- ดันแท็บ Event History ขึ้นมาเด่น + ปรับ breakpoint ให้เข้ากับ iPad/แท็บเล็ต

### Registration
- ล็อกการยกเลิกลงทะเบียนหลังเลยเวลาปิดรับ + เพิ่ม confirm ก่อน un-register

### Mobile / UI
- ฟอร์มประเมินในหน้า history: เลื่อนได้เต็มจอแม้มีดาวให้คะแนน, หัวฟอร์มกระชับบนมือถือ, รีเซ็ตการเลื่อนต่อ section
- โมดัล Manage User (หน้า admin students): จัด layout เป็น flex column + `maxHeight: 90vh` + body เลื่อนได้ ไม่ล้นจอบนมือถือ, padding/radius แบบ `clamp()`, header/footer ไม่ยุบ
- เพิ่มช่องแก้ **prefix** (native select) ข้างชื่อเต็ม และทำให้ API `PATCH /api/admin/users/[id]` รับ/บันทึก/audit ค่า prefix ด้วย
