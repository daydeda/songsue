## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **กิจกรรมหลายวันเช็คอินได้ทุกวัน** — กิจกรรมอย่าง CAMT LINK ที่จัดสองวันขึ้นไปสามารถบันทึกการเข้าร่วมแยกตามวันได้แล้ว ไม่นับรวมกันเป็นครั้งเดียวอีกต่อไป
- **ลงทะเบียนครั้งเดียว เข้าได้ทุกวัน** — โหมด "ลงทะเบียนครั้งเดียว" (once) ให้นักศึกษาลงทะเบียนกิจกรรมหนึ่งรอบแล้วสามารถเช็คอินได้ทุกวันที่จัด
- **โควตา Walk-in รีเซ็ตทุกวัน** — สำหรับกิจกรรมที่ตั้งค่าแบบ per_session แต่ละวันมี walk-in quota เป็นของตัวเอง วันใหม่โควตาก็เปิดใหม่
- **คะแนนบ้านนับแบบยุติธรรม** — นักศึกษาที่เข้าร่วมหลายวันนับคะแนนให้บ้านครั้งเดียว ไม่นับซ้ำตามจำนวนวัน
- **ปุ่มเข้าร่วมกลุ่ม LINE บ้าน** — เพิ่มปุ่มลิงก์เชิญเข้ากลุ่ม LINE ของแต่ละบ้านบนหน้า Leaderboard และหน้ารวมสมาชิก เพื่อให้นักศึกษาติดต่อสื่อสารกันได้สะดวกขึ้น
- **รายชื่อสมาชิกบ้านดูง่ายขึ้น** — หน้าแสดงสมาชิกของแต่ละบ้านรองรับการแบ่งหน้า (Pagination) ทีละ 50 คน ช่วยลดเวลาโหลด โดยหน้าแรกยังคงแสดงโพเดียม Top 3 ไว้อย่างสวยงาม
- **ปรับเฉดสีบ้านและโลโก้มาสคอตใหม่** — อัปเดตเฉดสีประจำบ้านให้ดูพรีเมียมขึ้น: สีแดง (มอม - Mom), สีเงิน/เทาอมฟ้า (โต - To), สีน้ำเงิน (หลวง - Luang), และสีเขียว (มะคอน - Makon) พร้อมมาสคอตที่วาดตกแต่งใหม่

## ฝั่งทีม (technical changelog)

### DB / Migration (ขั้นตอน 34–40 · idempotent, non-destructive)
- **step 34** — สร้างตาราง `event_sessions` ใหม่ (`id`, `event_id`, `title`, `start_time`, `end_time`, `sort_order`, `quota_walk_in`) พร้อม index บน `event_id` และ `(event_id, sort_order)`
- **step 35** — เพิ่มคอลัมน์ `events.registration_mode text NOT NULL DEFAULT 'once'` — event เดิมทั้งหมด default เป็น `'once'` พฤติกรรมไม่เปลี่ยน
- **step 36** — เพิ่ม `attendance.session_id uuid` แบบ NULLABLE ก่อน เพื่อให้ row เดิมยังถูกต้องระหว่าง backfill
- **step 37** — backfill `event_sessions`: สร้าง session เดียวต่อ event ที่ยังไม่มี session โดยคัดลอก `start_time`/`end_time`/`quota_walk_in` จาก event — idempotent ด้วย `WHERE NOT EXISTS`
- **step 38** — backfill `attendance.session_id` ให้ชี้ไปยัง session แรกสุด (ตาม `sort_order`, `start_time`) ของแต่ละ event
- **step 39** — สลับ unique constraint จาก `(event_id, student_id)` → `(session_id, student_id)` แบบ add-new-before-drop-old เพื่อไม่ให้มีช่วงว่างที่ scan ซ้ำได้; constraint เดิมลดเป็น non-unique index สำหรับ report/winner roll-up ที่ยังต้องการ query ตาม event
- **step 40** — บังคับ `attendance.session_id SET NOT NULL` หลัง backfill เสร็จ — ถ้า step 38 ตกหล่น row ใด ขั้นนี้จะ fail ดัง ๆ ก่อน deploy

### Schema & Database Script
- `eventSessions` table + Drizzle relations (`eventsRelations.sessions`, `eventSessionsRelations`, `attendanceRelations.session`)
- `attendance` unique index เปลี่ยนเป็น `idx_attendance_session_student` (session_id, student_id); เพิ่ม non-unique `idx_attendance_event_student` คู่ขนาน
- เพิ่มสคริปต์ `scripts/recolor-houses.mjs` สำหรับใช้รันอัปเดตสีบ้านใน DB ระดับ Production แบบ idempotent

### สิทธิ์เข้าถึง / PDPA & RBAC
- **ยกระดับความปลอดภัย PDPA**: เพิ่มหน้าสิทธิ์ดู Attendance (ดูผู้เข้าร่วม) แบบ Read-only ให้แก่ SMO, Club President และ Major President โดยจำกัดไม่ให้เห็นเบอร์โทรศัพท์, ข้อมูลการแพทย์ หรือผู้ติดต่อฉุกเฉินอย่างเด็ดขาด (API กรองทิ้งฝั่ง Server-side)
- `meds_check_option` ยังคงอยู่ที่ระดับ attendance row (ต่อ session) — registration staff เห็นแค่สัญญาณ ไม่เห็นรายละเอียด เหมือนเดิม; ไม่มีการเปิดช่องรั่วใหม่จาก session layer
- walk-in quota ต่อ session ถูก lock ที่ `event_sessions` row ด้วย `SELECT FOR UPDATE` แทน `events` row เดิม — ปิด TOCTOU window ระดับ session โดยไม่บล็อก session อื่น

### Scanner & Attendance UI
- **Custom Day/Session Dropdown** — เปลี่ยน Selector วันที่ของการสแกนจาก native `<select>` มาเป็นปุ่ม Dropdown คัสตอมแบบเช็คบ็อกซ์สวยงาม แสดงข้อความ "Day N — <Date>" พร้อมปิดอัตโนมัติเมื่อกดภายนอก
- **Wrap Event Title** — ปรับชื่อกิจกรรมยาว ๆ ในหน้า Attendance Modal ของแอดมินให้หักบรรทัดลงมาอย่างถูกต้อง (ไม่ตัดข้อความทิ้ง) ด้วย `whiteSpace: normal` + `break-word`
- `src/app/admin/scanner/page.tsx` — เพิ่ม session selector (calendar icon) แสดงเฉพาะกิจกรรมที่มีมากกว่า 1 session; default เป็น session "วันนี้" (window ครอบ now → upcoming → ล่าสุด)
- `sessionIdRef` mirror sessionId ใน ref เพื่อให้ camera decode callback ที่ register ครั้งเดียวส่ง session ถูกต้องเสมอแม้ staff สลับวัน
- `src/app/api/admin/scan/route.ts` — รับ `sessionId` เสริม (optional); ถ้าไม่ส่งมาให้ server resolve ด้วย `EventsService.resolveCurrentSessionId()`
- `scanner.service.ts` — `processScan()` รับ `sessionId`; ตรวจว่า session ต้องเป็นของ event นั้น (ป้องกัน cross-event sessionId ที่แต่งมือ); audit log ระบุชื่อวัน (เช่น "Day 1")

### Registration & Houses
- **Join LINE Group & Pagination** — เพิ่ม i18n key `joinLineGroup` ทั้ง 4 ภาษา (EN/TH/MM/CN), หน้าแสดงสมาชิกบ้านจัดกลุ่มหน้าละ 50 สมาชิก มีโพเดียมเฉพาะหน้า 1
- **Recolor Constants** — แก้ไขสีประจำบ้านและ gradient ต่างๆ ในโค้ด DashboardClient, digital-id page, แอดมินแดชบอร์ด และ globals.css ให้สอดคล้องกับมาสคอตใหม่
- `POST /api/events/[id]/register` — anchor registration row ไปที่ `firstSession.id`; single-session event พฤติกรรมเดิม; multi-day 'once' event จะให้ scanner สร้าง attended row ต่อ session เมื่อเช็คอินแต่ละวัน
- `scanner.service.ts` — เพิ่ม path "once-mode, first scan at THIS session": ถ้ามี event-level registration แต่ยังไม่มี session row ให้ insert attended row สำหรับ session นั้น (ON CONFLICT DO NOTHING ป้องกัน race)

### Admin Event Form
- `src/app/admin/events/page.tsx` — เพิ่ม registration mode toggle ("ลงทะเบียนครั้งเดียว" / "แยกแต่ละวัน") และ sessions/days editor (เพิ่ม/ลบ/แก้ชื่อวัน-เวลา-walk-in quota); สร้าง event ใหม่จะ auto-seed 1 session; เปิด form edit จะโหลด sessions เดิมมา prefill พร้อม id เพื่อให้ PUT update แทน recreate
- เมื่อสร้าง event แบบ single-day session row แรกจะ mirror ตาม start/end อัตโนมัติ ไม่ต้องกรอกซ้ำ

### Report / Export
- `GET /api/admin/events/[id]/attendance` — รับ query param `?sessionId=` เพื่อกรอง roster ตาม session; join session metadata ทุก row
- `GET /api/admin/events/[id]/report` (CSV) — เพิ่ม column "Session" (e.g. "Day 1") ต่อ row
- `GET /api/admin/events/[id]/export` (XLSX) — เพิ่ม column "Session" ใน header + ทุก data row

### House Points
- `src/lib/award-points.ts` — นับ distinct student ต่อบ้าน (`Set<studentId>`) ข้าม session เพื่อให้นักศึกษาที่เข้าหลายวันนับคะแนนบ้านครั้งเดียว; single-session event พฤติกรรมเดิม

### i18n / Shared
- `src/lib/event-schema.ts` — schema ใหม่ `sessionInputSchema` (zod) แชร์ระหว่าง POST/PUT events
- `src/lib/i18n.ts` — เพิ่ม key ใหม่ 4 ภาษา (EN/TH/MY/CN): `scanDayLabel`, `scanSessionLabel`, `registrationModeLabel`, `registrationModeOnce`, `registrationModePerSession`, `sessionsHeading`, `addDay`, `removeDay`, `sessionTitleLabel`, `sessionWalkInQuota`
- `EventsService.resolveCurrentSessionId()` — logic เลือก session ปัจจุบัน (same as client-side `pickCurrentSessionId`) ให้ server กับ UI เลือกวันตรงกัน
