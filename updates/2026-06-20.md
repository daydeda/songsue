## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **แจ้งเตือนแบบทดสอบ Pre-Test ค้างส่ง** — ถ้าลงทะเบียนกิจกรรมที่มี Pre-Test แล้วยังไม่ได้ทำ จะเด้งขึ้นมาเป็น modal + toast บนหน้า Dashboard และหน้า Digital ID โดยอัตโนมัติ แจ้งเตือนนี้จะไม่หายเองจนกว่าจะกดเปิดลิงก์ไปทำ
- **แบนเนอร์ "ฟอร์มที่ต้องทำ" บน Dashboard** — แสดงรายการฟอร์มทุกประเภทที่ยังค้างอยู่ (Pre-Test, Post-Test, ประเมินกิจกรรม) พร้อมชื่อกิจกรรมและจำนวนรายการค้าง กดได้เลย ลิงก์ตรงไปหน้าฟอร์มนั้น ๆ
- **แสกนเนอร์เตือนแอดมินด้วย** — หลังสแกน QR เช็คอินให้นักศึกษา ถ้ากิจกรรมนั้นมี Pre-Test ที่นักศึกษายังไม่ได้ส่ง จะมีการ์ดสีชมพู-แดงขึ้นพร้อม QR code และลิงก์ตรงไปฟอร์มนั้น เพื่อให้เจ้าหน้าที่แจ้งนักศึกษาได้ทันที

## ฝั่งทีม (technical changelog)

### DB / Migration
- **ไม่มีการเปลี่ยน schema และไม่มี migration** — feature นี้ derive ข้อมูลทั้งหมดจากตาราง `attendance`, `forms`, `formSubmissions` ที่มีอยู่แล้ว

### PDPA / สิทธิ์เข้าถึง
- ฟอร์มที่โหลดผ่าน `GET /api/forms/pending` อ่านแค่ metadata (form type, title, event name) — ไม่แตะข้อมูลการแพทย์ ไม่ต้องบันทึก audit log
- Banner และ notification synthesize ข้อมูลเฉพาะของ student ที่ล็อกอินอยู่เท่านั้น ไม่มี cross-user data exposure

### Forms / Notifications
- **`GET /api/forms/pending`** (route ใหม่) — query forms ที่ student ยังไม่ได้ submit โดยใช้ logic เดียวกับ submit gate จริง: K_pre ต้องการแค่การลงทะเบียน; K_post และ A (feedback) ต้องการ attendance จริง; S (skill) เป็นของ evaluator ไม่นำมาแสดง
- **`src/components/FormsDueBanner.tsx`** (component ใหม่) — banner mobile-first แสดงจำนวนรายการค้าง, type badge (Pre-Test / Post-Test / ประเมิน), ชื่อฟอร์ม, ชื่อกิจกรรม พร้อม deep-link ต่อรายการ; ใช้สี rose (`#e11d48`)
- **`/api/notifications/route.ts`** — synthesize notification ประเภท `pre_test_reminder` จาก K_pre forms ที่ค้างส่ง; set `autoDismiss: false` เพื่อให้ modal + toast ค้างอยู่จนกว่านักศึกษาจะกด
- **`src/components/NotificationModal.tsx`** — รองรับ type `pre_test_reminder` แสดง modal พร้อมลิงก์และข้อความ "แบบทดสอบ Pre-Test"
- **`src/components/NotificationToasts.tsx`** — render toast สำหรับ `pre_test_reminder` แบบ non-auto-dismiss
- **`src/lib/useNotifications.ts`** — ปรับ hook ให้เก็บ state ของ notification ที่ `autoDismiss: false` ไว้แยก ไม่ตั้ง timer dismiss

### Scanner & Attendance UI
- **`src/app/admin/scanner/page.tsx`** — หลัง scan สำเร็จ (check-in หรือ walk-in) ตรวจสอบ pending pre-test ของนักศึกษาคนนั้น; ถ้ามีให้แสดงการ์ด rose warning พร้อม QR code + ลิงก์ไปฟอร์ม; การ์ดหายเองเมื่อ scan รายถัดไป
- **`src/modules/events/scanner.service.ts`** — เพิ่ม `getPendingPreTest(eventId, studentId)` คืน form metadata ถ้ายังไม่ submit; ใช้ query เดียวกับ pending API

### Dashboard
- **`src/app/dashboard/DashboardClient.tsx`** — mount `FormsDueBanner` และ trigger `pre_test_reminder` notifications เมื่อ load
- **`src/app/dashboard/id/page.tsx`** — แสดง `pre_test_reminder` notification บน Digital ID page ด้วย

### i18n
- `src/lib/i18n.ts` — เพิ่ม key ใหม่ครบ 4 ภาษา (EN/TH/MM/CN): `formsDueBannerTitle`, `formsDueBannerItem`, `formsDueBannerEmpty`, `preTestReminderTitle`, `preTestReminderBody`, `preTestReminderLink`, `scannerPreTestWarningTitle`, `scannerPreTestWarningBody`
