## ฝั่งทีม (technical changelog)

### DB / Migration

- **ไม่มีการเปลี่ยน schema และไม่มี migration** — `src/db/schema.ts` และ `src/db/migrate.ts` ไม่ถูกแตะ; prod Supabase ไม่มีอะไรเปลี่ยน
- เพิ่ม npm scripts `db:migrate:container` / `db:seed:container` ใน `package.json` — เหมือน `db:migrate`/`db:seed` แต่ไม่ใช้ `--env-file` เพราะ container อ่าน `DATABASE_URL` จาก env ที่ Swarm stack inject ให้เลย; ใช้รันจาก Portainer web console แทน SSH

### PDPA / สิทธิ์เข้าถึง

- ไฟล์ PDPA (ฟอร์มสมัคร, สลิปการชำระเงิน) ที่เดิมอยู่ใน Supabase private bucket จะย้ายมาเก็บใน named volume `uploads-private` บน Swarm — volume นี้ไม่ publish port และไม่ expose ออก LAN; code gate (`src/lib/image-upload.ts`) และ auth-guard บน `/api/upload` ไม่มีอะไรเปลี่ยน
- `Dockerfile` เพิ่ม `mkdir -p .uploads-private && chown nextjs:nodejs .uploads-private` เพื่อให้ directory นี้มีสิทธิ์ถูกต้องก่อน volume mount

### Deploy / Infrastructure

- **`docker-stack.yml`** (ไฟล์ใหม่) — Portainer/Docker Swarm stack สำหรับ `dev2.camt.cmu.ac.th`; ใช้ pre-built image จาก GHCR; port 10780 → 3000 (CAMT reverse proxy routing ไปที่ `camtplaybase.camt.cmu.ac.th`); named volumes 3 ตัว: `uploads` (public โปสเตอร์/avatar), `uploads-private` (PDPA), `pg-data` (Postgres); DB ไม่ publish port — accessible เฉพาะ `web` service ผ่าน internal overlay network
- **`.github/workflows/docker-publish.yml`** (ไฟล์ใหม่) — GitHub Actions build Dockerfile และ push ไปยัง `ghcr.io/daydeda/smocamt-website` ทุกครั้งที่ push ไป `main`; tag `latest` + `<commit-sha>`; หลัง workflow เสร็จให้ไป Portainer → "Pull latest image version" เพื่อ deploy รอบใหม่
- **`Dockerfile`** — copy `src/`, `scripts/`, `elevate-admin.ts`, `tsconfig.json` เข้า runtime image เพื่อให้รัน migration / seed / elevate-admin ได้จาก Portainer web console (Swarm ไม่มี SSH); แก้ `.dockerignore` ให้กัน `node_modules` ที่ `npm ci` build ไว้ไม่ให้ถูก `COPY . .` ทับ
- **`docker-compose.yml`** — ติด comment "local dev only"; ลบ `.env` runtime mount ที่เสี่ยงต่อการ leak secrets
- **`university_server_deployment_guide.md`** — ติด deprecation banner (คู่มือ SSH/VM เก่าใช้ไม่ได้กับ Portainer Swarm แล้ว)

### Cutover Scripts (one-off)

- **`scripts/migrate-supabase-files.mjs`** — ดาวน์โหลด Supabase Storage ทั้ง 3 bucket (`uploads` = public โปสเตอร์/avatar, `form-uploads` = ไฟล์ฟอร์ม PDPA, `slips` = สลิปชำระเงิน) เข้า local volumes ของ container; รันครั้งเดียวตอน cutover จาก Portainer console
- **`scripts/rewrite-storage-urls.mjs`** — rewrite Supabase public URL (`https://*.supabase.co/storage/...`) ในฐานข้อมูลให้เป็น `/uploads/...`; idempotent (รันซ้ำปลอดภัย); hard-refuse ถ้า `DATABASE_URL` ยังชี้ไป Supabase เพื่อป้องกัน rewrite prod โดยไม่ตั้งใจ
