## ฝั่งทีม (technical changelog)

### DB / Migration

- **ไม่มีการเปลี่ยน schema และไม่มี migration** — `contactChannels` เป็นคอลัมน์ที่มีอยู่แล้วใน `users`; เพิ่มแค่การ select และ map ออกมาในไฟล์ export

### PDPA / สิทธิ์เข้าถึง

- `contactChannels` ถือเป็นข้อมูลติดต่อทั่วไป (LINE / Instagram ฯลฯ) — **ไม่ใช่ข้อมูลการแพทย์** จึงไม่ได้อยู่ใน `canViewMedical` block และไม่ต้องบันทึก audit log
- สิทธิ์การ export ไม่เปลี่ยน: ยังคงจำกัดเฉพาะ `admin` และ `super_admin` ตามเดิม

### Events / Export

- **ปัญหาเดิม** — ไฟล์ attendee `.xlsx` ที่ดาวน์โหลดจากหน้า admin กิจกรรมขาดคอลัมน์ Contact Channels ไปทั้งหมด ทั้งที่ submissions export มีคอลัมน์นี้แล้ว ทำให้ข้อมูลสองชุดไม่ตรงกัน
- **`src/app/api/admin/events/[id]/export/route.ts`** — เพิ่ม `contactChannels` ใน Drizzle select query, เพิ่ม column header `"Contact Channels"` ต่อจาก `"Phone"` ในแถว header array, และ map ค่าออก (`u?.contactChannels || ""`) ลงใน row object ที่สร้าง XLSX; type `AttendeeUser` ได้รับ field `contactChannels?: string | null` ด้วย
- attendee export และ submissions export มีคอลัมน์ตรงกันแล้ว
