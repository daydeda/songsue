ช่วง: 24 - 25 มิ.ย. 69 · สรุปไว้สำหรับลง Discord และให้ทีมที่เกี่ยวข้องอ่าน

## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **ระบบปฏิทินกิจกรรมรายเดือน (Month-view Calendar) และการเชื่อมต่อปฏิทินภายนอก (.ics)**
  - เพิ่มหน้าปฏิทินใหม่ (`/dashboard/calendar`) แสดงรายการกิจกรรมและบันทึกปฏิทิน (Calendar Entry) ที่ผู้เรียนมีสิทธิ์เข้าร่วม
  - รองรับการคัดลอกลิงก์เพื่อนำไปสมัครรับข้อมูล (Subscribe Feed) บน Google Calendar, Apple Calendar หรือ Outlook ได้ทันที พร้อมระบบรักษาความปลอดภัยตามมาตรฐาน PDPA ที่ช่วยให้รีเซ็ตหรือยกเลิก token (.ics feed token) ได้ตลอดเวลา
  - ปรับปรุงการแสดงผลโปสเตอร์และรายละเอียดที่เป็นข้อความจัดรูปแบบ (Rich-Text) ใน Modal แสดงรายละเอียดของปฏิทิน
- **ระบบไทม์ไลน์หน้าแดชบอร์ดและแถบสถิติสะสม (Timeline View & Overview Stats)**
  - เพิ่มแถบควบคุมให้สามารถสลับการแสดงผลของหน้ากิจกรรมในหน้าแดชบอร์ดหลักระหว่างแบบ "การ์ดกิจกรรม (Cards Grid)" และ "ลำดับเวลา (Timeline agenda)" โดยแบ่งเป็นหมวดหมู่ตามช่วงเวลา (Today / This Week / Upcoming / Past) ทำให้ดูรายการกิจกรรมในอดีตได้สะดวกขึ้น
  - เพิ่มแถบแสดงสถิติภาพรวม (Upcoming / This Week / Attended / Points) ของผู้ใช้งานที่ Logged in เพื่อแสดงคะแนนบ้านสะสมที่มาจากการเข้าร่วมกิจกรรมแบบ Real-time
- **หน้าแจ้งเตือนการย้ายโฮสต์เซิร์ฟเวอร์เก่า (Retired Vercel Notice)**
  - สำหรับผู้ใช้งานที่ยังเข้าผ่านลิงก์ Vercel เดิม ระบบจะขึ้นหน้าจอแจ้งเตือน bilingual (TH/EN) "We've moved" ชี้ไปยัง URL ใหม่ที่ย้ายขึ้นเซิร์ฟเวอร์ CAMT เรียบร้อยแล้ว เพื่อป้องกันปัญหาข้อมูลกระจายไปยัง database เก่า (Supabase)

## ฝั่งทีม (technical changelog)

### DB / Migration
- **ตารางใหม่ (`calendar_entries` และ `calendar_feed_tokens`):** รองรับระบบบันทึกปฏิทินที่แยกจากกิจกรรมหลัก และรองรับความปลอดภัยในการดึง feed ปฏิทินรายคน (Migration step 42-43; add-only, idempotent)
- **ประวัติคะแนนฟอร์มประเมิน (`score_history.form_id`):** เพิ่ม nullable FK ไปยังตาราง `forms` เพื่อติดตามคะแนนที่ได้จากฟอร์มประเมิน ป้องกันความสับสนกับประวัติคะแนนของตัวกิจกรรมหลัก (Migration step 44)

### PDPA / สิทธิ์เข้าถึง
- **ความปลอดภัยของ .ics Feed Token:** Token สำหรับดึงปฏิทินทำหน้าที่เสมือน API Key ส่วนบุคคลของนักศึกษา โดยเส้นทาง `/api/calendar/feed/[token]` จะข้าม proxy middleware (ซึ่งปกติบล็อก /api) แต่มีการตรวจสอบ token ความปลอดภัยแบบ in-route validation ป้องกันข้อมูลสิทธิ์รั่วไหล
- **การลบข้อมูลส่วนบุคคล (PII Deep Redaction):** ตัว Logger ตัวใหม่ (`src/lib/logger.ts`) จะลบฟิลด์ที่อ่อนไหว เช่น `address`, `medicalInfo`, `contactChannels` ฯลฯ ในระดับ nested object ก่อนบันทึกหรือส่ง Log ไปยัง Discord webhook

### Events & Forms / สแกนเนอร์
- **ระบบนับจำนวนผู้เช็คอินแบบ Real-time:** เพิ่ม `/api/admin/scan/count` และแก้ไข Scanner client ให้อ่านเฉพาะแถว attendance ที่มีสถานะ `'attended'` เท่านั้น (ไม่นับรวมผู้สมัครล่วงหน้าแต่ยังไม่มาเช็คอินจริง) เพื่อให้ยอดเช็คอินบนแผงสแกนเนอร์ตรงตามผู้เข้าร่วมจริง
- **การเปิดฟอร์มประเมินอีกครั้งและการคืนคะแนน (Points Clawback):** แอดมินสามารถเปิดฟอร์มที่มอบคะแนนไปแล้วใหม่อีกครั้งได้ โดยระบบจะรัน transaction ลบคะแนนสะสมของบ้านและลบแถว ledger ใน `score_history` ของฟอร์มดังกล่าวออกโดยอัตโนมัติ เพื่อเตรียมรับคะแนนรอบใหม่เมื่อปิดฟอร์มอีกครั้ง
- **การนับ Distinct People ในหน้าสรุปเช็คอิน:** แก้ไขหน้าสรุปประวัติเช็คอินในแอดมินบอร์ด ให้อิงจำนวนจาก `attendanceUnits` (Distinct people) แทนความยาวอาร์เรย์รวม เพื่อไม่ให้เกิดการนับยอดคนซ้ำ (Double counting) ในกิจกรรมแบบหลายวัน (Multi-day events)
- **ระบบแปลภาษาแผง Form Builder:** จัดทำคีย์แปลภาษาหน้า Form Builder 4 ภาษาหลัก (TH, EN, MM, CN) แทนภาษาอังกฤษแบบฮาร์ดโค้ดเดิม

### Deploy / Infrastructure
- **การแก้ไข Docker Build ใน CI:** เปลี่ยนการ Copy โฟลเดอร์ build จาก `.next.nosync` (ซึ่งเกิดขึ้นจากการปรับแต่งเครื่อง Local ที่ใช้ iCloud Drive) กลับมาใช้ `.next` ตามปกติ เพื่อให้ระบบ CI/CD บน GitHub Actions สามารถ Build Image ได้สำเร็จ
- **Portainer GitOps & Auto-Deploy (CD Loop):** ตั้งค่า CD บน Actions ให้ส่ง Webhook ยิงไปที่ Portainer ทุกครั้งที่ Merge โค้ดเข้า `main` เพื่อสั่ง Portainer pull image จาก GHCR และ redeploy stack แบบ GitOps อัตโนมัติ รวมถึงย้าย credentials และ secrets ใน `docker-stack.yml` ไปกรอกเป็น Environment variables บน Portainer แทนการใส่ใน repo
- **รูปภาพอัปโหลดหลังย้ายเซิร์ฟเวอร์ (Media Proxy Streaming):** เนื่องจาก Node/Next.js ใน container ไม่สามารถเสิร์ฟรูปภาพที่อัปโหลดขึ้นมาในระดับ runtime ได้โดยตรง (หลังย้ายออกจาก Supabase Storage) ระบบได้สร้าง API Endpoint `/api/media/[name]` คอยอ่านไฟล์รูปโปสเตอร์/สินค้าจากดิสก์ และนำกฎ `/uploads/:name` rewrite ผ่าน `next.config.ts` ชี้เข้า API ตัวใหม่นี้ พร้อมตั้งค่า Long-caching บน Nginx และ Proxy

### Performance / Performance Tuning
- **DB Connection Pooling:** ปรับเพิ่ม connection pool size ผ่าน env `DB_POOL_MAX` เป็น 15 (จากเดิม 5 ของยุค Supabase) เพื่อให้เหมาะสมกับการคุยกับ Postgres Container ตรง ๆ
- **App-layer Caching (unstable_cache):** ตั้งค่าแคชข้อมูลสถิติบอร์ดและคะแนนบ้านเป็นเวลา 15 วินาทีใน endpoint ยอดฮิตอย่าง `/api/houses`, `/api/houses/activity`, `/api/houses/individual` และ `/api/announcement` เพื่อตัดภาระการ query ฐานข้อมูลตรง ๆ ทุกรอบที่มีการ polling หน้าจอ ปรับปรุงให้รองรับ concurrency สูงในวันจัดกิจกรรม
- **Leaderboard Cache Invalidation:** เพิ่ม revalidateTag ใน endpoint การปรับคะแนนแบบ manual ทำให้เมื่อแอดมินแก้ไขคะแนนบ้าน หน้าแสดงผลจะล้างแคชและอัปเดตเป็นปัจจุบันทันที

### Tests & CI Gates
- **ระบบทดสอบ Vitest:** ติดตั้ง Vitest และจัดทำ Unit test รวม 96 ตัว ครอบคลุมการเช็คสิทธิ์แอดมิน/สแกนเนอร์, ความแม่นยำของ QR Token และความถูกต้องของ Audit Log Hash-chain
- **CI Workflow Gate:** บังคับรันตรวจสอบ test และ build ใน PR ก่อนที่จะ merge เข้าสู่ `main` เสมอ

### Docs / Wiki
- **GDD (Game Design Documents):** นำเสนอแนวคิดระบบ P2P Battle Platform ร่วมกับการใช้ WebRTC Signaling Flow และพัฒนาเอกสารกติกาการสู้รบ/การจัดห้อง และคู่มือ OX Game Module ใหม่ทั้งหมด
- **Portainer Migration Path:** เพิ่มคู่มือการอัปเกรดฐานข้อมูลและการรัน migrations บน container สำหรับโปรดักชันเซิร์ฟเวอร์ใหม่
