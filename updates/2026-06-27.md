### ความปลอดภัย / Security & Validation

- **UUID validation ก่อน DB query** — route ทุกตัวที่รับ `[id]` parameter (events, calendar, attendance, export, form, students, users) เพิ่ม `z.string().uuid()` parse ก่อนส่ง query; PostgreSQL invalid UUID (error 22P02) ที่เคยทำให้ได้ 500 กลายเป็น 400 พร้อม message ชัดเจน

- **รองรับชื่อ field และตัวเลือกภาษาไทย + อังกฤษยาวขึ้น** — ช่องกรอกข้อมูลเฉพาะสินค้า (เช่น "ชื่อบนเสื้อ / Name on back") รองรับ label และตัวเลือกที่มีทั้งภาษาไทยและอังกฤษในบรรทัดเดียวกันได้อย่างสมบูรณ์ — ข้อความยาวจะ wrap แทนที่จะล้นออกนอกกรอบ ทั้งในหน้าสั่งซื้อ, หน้าสรุปก่อนจ่าย และในหน้า Admin ดู order

## ฝั่งทีม (technical changelog)

### Shop / Product Schema

- **`src/lib/shop-product-schema.ts`** (+3 บรรทัด) — เปลี่ยน `imageUrls: z.array(z.string().url())` เป็น `z.array(z.string())` — root cause: server self-hosted (`/api/upload`) คืน relative path เช่น `/uploads/x.jpg` ซึ่ง `z.string().url()` reject เพราะไม่มี scheme; events ใช้ `z.string()` bare มาตลอด — align ให้ตรงกัน; กระทบทั้ง create และ edit product

### Shop / Custom Fields

- **`src/lib/shop-custom-fields.ts`** (+4 บรรทัด, −3 บรรทัด) — เพิ่ม cap ของ bilingual content ทั้งสามจุด:
  - `label`: max 60 → **1,000** ตัวอักษร (schema + comment อธิบาย rationale)
  - `options[]`: max 60 → **1,000** ตัวอักษรต่อ option
  - `maxLength` (max cap ของ text field): max 500 → **1,000** — ทุก change เป็น Zod schema ฝั่ง server ซึ่งทั้ง admin product API และ order POST ใช้ร่วมกันผ่าน `validateCustomAnswers()`

### Admin / Shop

- **`src/app/admin/shop/AdminShopClient.tsx`** (+4 บรรทัด, −3 บรรทัด) — 3 จุดใน product form editor:
  - label input: เพิ่ม `maxLength={1000}` เพื่อ cap ที่ client
  - max-length number input: เพิ่ม `max={1000}` และ `Math.min(1000, ...)` clamp เพื่อ cap ที่ client
  - client-side guard ใหม่: validate options ทุกตัวว่าไม่เกิน 1,000 ตัวอักษรก่อน submit (ข้อความ error ครบ TH/EN)
  - admin order custom values (`AdminOrderRow`): เพิ่ม `overflowWrap: "anywhere", wordBreak: "break-word"` บน `<span>` — ป้องกัน long/unbroken strings ล้นออกนอก order card

### Shop / Storefront — Dropdown Fix

- **`src/app/dashboard/shop/ShopClient.tsx`** (+7 บรรทัด, −3 บรรทัด) — `CustomSelect` component scroll handler เดิม (`window.addEventListener("scroll", close, true)`) ปิด dropdown บน **ทุก** scroll event ที่ capture ได้รวมถึง scroll ภายใน menu; แก้โดยเปลี่ยนเป็น `onScroll` handler ที่ตรวจ `menuRef.current?.contains(e.target as Node)` — ถ้า scroll event originate จากภายใน menu จะ `return` ออกโดยไม่ปิด; ยัง close บน resize, Escape, click นอก dropdown, และ scroll จาก ancestor (modal body) ตามปกติ; presentation-only ไม่กระทบ API หรือ order logic

### Shop / Storefront — Display Hardening

- **`src/app/dashboard/shop/ShopClient.tsx`** (ต่อเนื่อง) — 2 จุดใน `ProductModal`:
  - custom field label (`<label>` render ของ input): เพิ่ม `overflowWrap: "anywhere", wordBreak: "break-word"` — wrap ข้อความยาวแทนการล้น
  - order summary rows (`customFields.filter(...).map(...)`): เพิ่ม style เดียวกัน — label และค่าที่กรอกใน summary ก่อนจ่ายเงิน wrap ได้ถูกต้อง

- **ไม่มี DB migration ใหม่** — fix ทั้งหมดอยู่ที่ application layer (Zod schema และ React component); ไม่มีการเพิ่มหรือเปลี่ยนคอลัมน์; ไม่ต้องรัน `db:migrate:container`
