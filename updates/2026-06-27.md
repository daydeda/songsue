### ความปลอดภัย / Security

- **แก้การปลอม IP (X-Forwarded-For)** — ทุก path ที่ดึง IP (`getClientIp` ใน `audit.service.ts`, `rate-limit.ts`, และ `auth.ts`) เปลี่ยนเป็น: prefer `X-Real-IP` ก่อน (nginx overwrite ด้วย `$remote_addr` จริง, ปลอมไม่ได้); fallback เป็น XFF hop **สุดท้าย** (nginx append); ไม่ใช้ `xff[0]` ซึ่ง client ควบคุมได้และจะสร้าง rate-limit bucket ใหม่ทุก request หรือ forge IP ใน audit log
- **Rate limiter ย้ายไปใช้ Postgres** (`src/lib/rate-limit.ts`) — เดิมเป็น in-memory Map ที่ reset ทุกครั้งที่ container รีสตาร์ทและแชร์ข้ามหลาย instance ไม่ได้; เปลี่ยนเป็น atomic upsert บน `rate_limit` table — window อยู่รอดการ deploy, ทำงานได้ถูกต้องหลาย instance; fail-open ถ้า DB error (rate limiter ไม่ควรบล็อก traffic ที่ถูกกฎหมาย)
- **Upload rate limit + size cap** (`src/app/api/upload/route.ts`, `src/lib/image-upload.ts`) — เพิ่ม rate limit 20 ครั้ง/นาที/IP บน `/api/upload` (sharp re-encode หนัก CPU บน single-instance); เพดานไฟล์ลงมาเป็น 5MB ทั้ง client route และ `hardenImageUpload()` helper (ก่อนหน้าตั้งเป็น 10MB โดยผิดพลาด — ไม่ตรง CLAUDE.md)
- **ตัด super_admin email hardcode ออก** (`src/auth.ts`) — `SUPER_ADMIN_EMAILS` env var ไม่มี hardcoded fallback อีกต่อไป (เดิมใส่ personal email ไว้เป็น default — config landmine); super_admin ที่มีอยู่ใน DB ยังคงอยู่ครบ; ตั้ง env var ผ่าน Portainer เพื่อให้ auto-promote ทำงาน
- **Assert `AUTH_URL` ในโหมด production** — `auth.ts` throw ทันทีที่ startup ถ้า `AUTH_URL` ไม่ได้ตั้งค่า (กัน OAuth callback host injection ผ่าน `Host` header เมื่อใช้ `trustHost:true`); ข้าม assert ระหว่าง `next build`
