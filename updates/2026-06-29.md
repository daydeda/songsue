## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **ส่งลิงก์ตรงไปยังกิจกรรมที่น่าสนใจให้เพื่อนได้แล้ว** — เปิด modal กิจกรรมใด ๆ แล้วกดปุ่ม **"คัดลอกลิงก์"** ที่มุมซ้ายล่างของหน้าต่าง; ลิงก์จะเปลี่ยนเป็นสีเขียวพร้อมข้อความ "คัดลอกแล้ว!" ยืนยัน; ส่งลิงก์ผ่าน LINE หรือ chat ใด ๆ — คนที่รับลิงก์กดเปิดแล้วจะเห็น modal กิจกรรมนั้นขึ้นมาทันที ไม่ต้องไล่หาเอง
- **ลิงก์ที่แชร์มาจาก SMO หรือจากเพื่อนเปิดได้ถูกต้อง** — ก่อนหน้านี้ลิงก์ `/dashboard?event=<id>` ที่แชร์มาบางครั้งเปิดหน้า Dashboard เปล่า ๆ โดยไม่มี modal ขึ้นมา ตอนนี้แก้แล้ว — modal เปิดตรงกิจกรรมนั้นเสมอ
- **ปุ่ม Back บนมือถือและเบราว์เซอร์ปิด modal แทนที่จะออกจากหน้า Dashboard** — เดิมกด Back หลังเปิด modal จะพากลับออกไปหน้าก่อนหน้าเลย; ตอนนี้กด Back ครั้งแรกปิด modal แล้วค่อยอยู่ที่ Dashboard ต่อไป
- **ปุ่ม "Copy link" รองรับทุก 4 ภาษา** — EN "Copy link" / TH "คัดลอกลิงก์" / CN "复制链接" / MM "လင့်ခ်ကူးယူ" พร้อมข้อความยืนยัน "Copied!" / "คัดลอกแล้ว!" / "已复制！" / "ကူးယူပြီး!" เมื่อสำเร็จ

## ฝั่งทีม (technical changelog)

### Dashboard / Shareable Event Preview

- **`src/app/dashboard/DashboardClient.tsx`** (+124, −27 บรรทัด) — เปลี่ยนสถาปัตยกรรมการติดตาม modal จาก `useState<Event | null>` มาเป็น `useState<string | null>` (event id) โดย `previewEvent` ถูก derive ระหว่าง render จาก `events.find()` — ทำให้ URL เป็น single source of truth และแก้ React lint error ที่มีอยู่เดิม (set-state-inside-useEffect จาก deep-link handler เดิม)
- **URL mirroring** — `openPreview(e)` เพิ่ม `window.history.pushState({ event: e.id }, ..., ?event=<id>)` ทำให้ Back button pop entry นี้และปิด modal; `closePreview()` มี 2 branch:
  - ถ้า `history.state?.event` มีค่า (เปิดผ่าน `openPreview`) → `window.history.back()` pop entry ที่ push ไว้; `popstate` listener รับแล้ว set `previewEventId = null`
  - ถ้า user ลงจุดที่ shared link โดยตรง (ไม่มี pushed entry) → `window.history.replaceState(null, ..., pathname)` strip param แล้ว set `previewEventId = null` ทันที — ไม่ navigate ออกจากหน้า
- **`useState` lazy initializer** — seed `previewEventId` จาก `window.location.search` ณ ตอน mount (SSR-guarded ด้วย `typeof window === "undefined"` check); ทำให้ deep-link เปิด modal ทันทีที่ events list โหลดเสร็จโดยไม่ต้องผ่าน `useEffect` scan เหมือนเดิม
- **`popstate` listener** — `useEffect` ที่ register `window.addEventListener("popstate", ...)` คอย mirror URL เข้า state ทุกครั้งที่ Back/Forward เปลี่ยน `?event=`; cleanup ใน return เพื่อไม่ให้ listener leak เมื่อ component unmount
- **`copyShareLink()`** — async helper ที่ build URL จาก `window.location.origin + pathname + ?event=<encoded id>`; ลอง `navigator.clipboard.writeText()` ก่อน; fallback เป็น hidden `<textarea>` + `document.execCommand("copy")` สำหรับ in-app webview (LINE, Facebook) ที่ไม่ได้รับ clipboard permission; ทั้งสอง path set `linkCopied = true` พร้อม `setTimeout` 2 วินาที reset
- **ปุ่ม "Copy link"** — วางที่ `marginRight: "auto"` ฝั่งซ้ายของ modal footer; icon สลับระหว่าง `Share2` (default) และ `Check` สีเขียว (เมื่อ copied); border และ color เปลี่ยนตาม state พร้อม CSS transition 0.2 s; import icon `Share2` และ `Check` เพิ่มเข้า lucide-react import list
- **Call sites** — `setPreviewEvent(e)` ทุกจุดเปลี่ยนเป็น `openPreview(e)` และ `setPreviewEvent(null)` ทุกจุดเปลี่ยนเป็น `closePreview()` ครอบคลุมทั้ง event card click, "Read more" button, timeline row, backdrop click, และ Close button

### DB / Migration

- **ไม่มี DB migration ใหม่** — pure client-side state/URL change; ไม่มีการเพิ่มหรือเปลี่ยนคอลัมน์; ไม่ต้องรัน `db:migrate:container`

### PDPA / สิทธิ์เข้าถึง

- **ไม่มีการเปลี่ยน access control, PDPA gating หรือ audit log** — feature อยู่ใน Dashboard UI layer ทั้งหมด; ไม่มี API ใหม่ ไม่มี read path ใหม่สำหรับ medical detail; gate ทุกชั้นยังคงอยู่ครบถ้วน

- **นักศึกษาปริญญาโท (KIM / DTM) และปริญญาเอก (KIM / DTM) สมัครสมาชิกได้ถูกต้องแล้ว** — หน้า Onboarding และหน้าแก้ไขโปรไฟล์ตรวจสอบหลักที่ 5 ของรหัสนักศึกษาเพื่อระบุระดับการศึกษา: ป.โท แสดง SE / KIM / DTM ให้เลือก; ป.เอก แสดง KIM / DTM; ป.ตรี ยังคงเป็น ANI / DG / DII / MMIT / SE เหมือนเดิม — ไม่ต้องพิมพ์เอง ไม่ต้องเลือกผิด
