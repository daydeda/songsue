## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **ระบบเล่นเกมท้าดวลเรียลไทม์ (P2P OX Battle)**
  - สามารถสร้างห้องเล่นเกม OX (Tic-Tac-Toe) และท้าดวลกับเพื่อนได้ผ่านการสแกน QR Code หรือกรอกรหัสห้อง 4 หลัก
  - หน้ากระดานเกมและระบบเวลาเทิร์น 60 วินาทีแบบเรียลไทม์ พร้อมระบบเช็คการแพ้/ชนะ/เสมอ และสตรีค (Streak) ในหน้าประวัติการดวลส่วนตัวและหน้า Leaderboard
- **การแชร์ลิงก์กิจกรรมตรง (Shareable Event Link)**
  - สามารถแชร์กิจกรรมที่สนใจให้เพื่อนได้โดยกดปุ่ม **"คัดลอกลิงก์"** (Copy Link) ที่อยู่มุมซ้ายล่างของ modal กิจกรรมนั้น ๆ
  - ลิงก์ที่แชร์จะเป็นรูปแบบ `/dashboard?event=<id>` เมื่อผู้อื่นคลิกเปิดจะแสดง modal กิจกรรมนั้นขึ้นมาทันที และการกดปุ่ม Back ของเบราว์เซอร์หรือบนมือถือจะเป็นการปิด modal โดยไม่เปลี่ยนหน้าเว็บ
- **แสดงช่วงเวลารับสมัครของกิจกรรม (Registration Window Display)**
  - เพิ่มการแสดงผลช่วงเวลาเปิด-ปิดรับสมัคร (Registration Window) บนการ์ดกิจกรรมและใน modal รายละเอียดกิจกรรม เพื่อให้นักศึกษาไม่พลาดเวลาลงทะเบียน
- **ระบบปฏิทินที่รองรับรายการแบบทำซ้ำ (Recurring Calendar Entries)**
  - ปฏิทินแสดงผลกิจกรรมที่เกิดขึ้นเป็นประจำ (Daily, Weekly, Monthly) ได้ถูกต้อง ช่วยให้วางแผนและติดตามกิจกรรมระยะยาวได้ง่ายขึ้น
- **ปรับปรุง Onboarding และโปรไฟล์สำหรับนักศึกษาบัณฑิตศึกษา (ป.โท/ป.เอก)**
  - หน้า Onboarding และหน้าแก้ไขโปรไฟล์จะตรวจสอบจากรหัสนักศึกษาโดยอัตโนมัติเพื่อแสดงตัวเลือกสาขาให้สอดคล้องกับระดับการศึกษา (ป.โท แสดง SE / KIM / DTM และ ป.เอก แสดง KIM / DTM)
  - ระบบจะรีเซ็ตค่าสาขาวิชาเป็นค่าเริ่มต้นให้โดยอัตโนมัติ หากผู้ใช้งานเปลี่ยนรหัสนักศึกษาแล้วส่งผลให้ระดับการศึกษาเปลี่ยนไป เพื่อป้องกันข้อมูลผิดพลาด

## ฝั่งเจ้าหน้าที่ / ผู้ดูแล (Admin / Staff UX)

- **ระบบป้องกันการเช็คชื่อผิดกิจกรรม (Live Event Reminder)**
  - ในหน้าเช็คชื่อ QR Scanner ระบบจะดึงกิจกรรมที่กำลังจัดอยู่ (Live) ณ ช่วงเวลานั้นขึ้นมาเป็นตัวเลือกเริ่มต้น (Default) โดยอัตโนมัติเมื่อโหลดหน้า
  - จัดเรียงลำดับให้กิจกรรมที่กำลัง Live อยู่ด้านบนสุดของรายการเลือกเสมอ
  - แสดงป้าย **"LIVE"** สีเขียวกะพริบพร้อมกับชื่อกิจกรรมและรายละเอียดวัน/รอบ (session) บนแถบด้านบนสุดของกล้องขณะกำลังสแกน
  - เพิ่มการ์ดแจ้งเตือนสีเหลืองอำพันพร้อมระบุทางเลือกในการเปลี่ยนกิจกรรมทันที หากหน้าจอกล้องระบุว่าเป็นกิจกรรมที่ไม่ได้กำลัง Live อยู่ในปัจจุบัน ป้องกันปัญหาเจ้าหน้าที่เช็คชื่อผิดกิจกรรม
- **แก้ปัญหากล้องกลับด้าน (Camera Mirroring Fixes)**
  - พัฒนาระบบปรับมุมกล้องให้ไม่กลับด้าน (un-mirror) เมื่อระบบต้อง fallback ไปใช้กล้องหน้า (เช่น การทดสอบบนโน้ตบุ๊ก หรืออุปกรณ์ที่ไม่มีกล้องหลัง)
  - พลิกภาพกล้องโดยใช้ CSS `transform: scaleX(-1)` อัตโนมัติทุกกรณีที่ตรวจพบว่าอุปกรณ์ที่ใช้สแกนรายงานค่า facingMode ไม่ใช่กล้องหลังตัวหลัก (`"environment"`)
- **ปรับปรุงหน้าค้นหาทะเบียนนักศึกษา (Admin Students Directory)**
  - ปรับแยกแถบค้นหา (Search Bar) ให้กว้างเต็มความกว้างหน้าจอแยกออกมาเป็นอิสระ เพื่อแก้ไขปัญหาช่องพิมพ์โดนเบียดบนหน้าจอขนาดเล็ก
  - เพิ่มตัวกรอง (Filter) ค้นหาตามระดับการศึกษา (Undergraduate / Master's / Ph.D.) และชั้นปีการศึกษา (ปี 1 ถึง ปี 5+) เพื่อช่วยให้เจ้าหน้าที่ค้นหากลุ่มเป้าหมายได้อย่างรวดเร็ว

## ฝั่งทีมพัฒนา (Technical / Infrastructure Changelog)

### ORM / Database Integration (Drizzle ORM & PGlite)
- ปรับเปลี่ยนโครงสร้างการเข้าถึงฐานข้อมูลมาใช้ **Drizzle ORM** ร่วมกับไดรเวอร์ **PGlite** (Local Postgres) และ Postgres ตัวเต็มสำหรับการรันงานจริง
- เพิ่มและปรับปรุงสคริปต์ Database Migration (`src/db/migrate.ts`) ให้สามารถสร้างตารางใหม่และนำเข้าโครงร่างฐานข้อมูล (Schema) แบบอัตโนมัติและปลอดภัย (Idempotent)
- จัดเตรียมเอกสารการตั้งค่า Drizzle ORM และคู่มือจัดการ Schema ใน `docs/software/03-data-schema.md` และสร้างตัวอย่างไฟล์กำหนดค่าที่ปลอดภัยสำหรับพัฒนาใน Local (`drizzle.config.ts`, `.env.production.example`)

### Custom NextAuth & JWT Session Management
- ปรับปรุงการล็อกอินเข้าสู่ระบบ (CMU Account) โดยปรับคอนฟิก NextAuth v5 ให้ทำงานร่วมกับ Drizzle Adapter และเก็บ Session บนฐานข้อมูล
- บูรณาการการถอดรหัสและจัดการ Session ผ่านกลไก JWT เพื่อเพิ่มประสิทธิภาพและความเสถียรในการประเมินสิทธิ์ใช้งาน

### Optimization & Security Hardening
- **Model Tiering**: ปรับลดระดับการใช้โมเดล AI ของทีมพัฒนา โดยใช้ Sonnet สำหรับงานเชิงโครงสร้างธรรมดา (เช่น route-gate-auditor, test-author) และ Haiku สำหรับตัวสร้างไฟล์เมล็ดพันธุ์ (seed-author) ช่วยควบคุมค่าใช้จ่าย API ขณะที่ยังคงใช้ Opus สำหรับความปลอดภัยและฐานข้อมูลเชิงลึก
- **Client-Side Image Compression**: เพิ่มระบบบีบอัดรูปภาพโปรไฟล์ฝั่งไคลเอนต์ก่อนการอัปโหลดขึ้นเซิร์ฟเวอร์ เพื่อป้องกันปัญหา HTTP 413 (Payload Too Large) ของ Reverse Proxy
- **Shop Surcharges & Pagination**: หน้าร้านค้ารองรับการบวกราคาเพิ่มแยกตามตัวเลือกสินค้าเฉพาะ (Price Surcharges) และเพิ่มระบบแบ่งหน้า (Pagination) ในหน้ารายการสินค้าและรายการสั่งซื้อของผู้ดูแลระบบ
- **Submissions List Improvements**: เพิ่มการแบ่งหน้าและการเรียงลำดับตามวันที่ส่งแบบประเมินผลล่าสุด (Newest First) บนหน้าประวัติการส่งฟอร์มของผู้ใช้และหน้ารายการสำหรับแอดมิน

### DB / Migration
- มีการเพิ่ม Migration ชุดใหม่สำหรับระบบเก็บข้อมูลเกม (ตาราง `game_rooms`, `game_moves`, `game_stats`, `webrtc_signals`) และตารางผู้ใช้ Drizzle สำหรับ NextAuth
- การอัปเกรด Schema ฝั่งร้านค้า (การจัดการ visibility, ตัวเลือกสินค้า และค่าจัดส่ง) ถูกควบรวมเรียบร้อยแล้วโดยไม่ต้องทำลายข้อมูลเดิม

### PDPA / สิทธิ์เข้าถึง
- ยังคงรักษาความปลอดภัยตามมาตรฐาน PDPA: หน้าแสดงข้อมูล Roster และประวัติเข้าร่วมกิจกรรมสำหรับบทบาท SMO/ประธานชมรม จะถูกกรองเบอร์โทรและข้อมูลการแพทย์ออกที่ระดับ Server-side เสมอ
- สำหรับระบบเกม P2P Battle ได้กำหนดให้ผู้ใช้ต้องล็อกอินและผ่านขั้นตอน Onboarding ก่อนจึงจะสามารถสร้างหรือเข้าร่วมห้องเกมได้ เพื่อป้องกันสแปมบ็อทและรักษาความถูกต้องของการบันทึกประวัติการเล่น
