## ฝั่งทีม (technical changelog)

### Routing / Landing Page

- **`src/app/page.tsx`** — เปลี่ยนจาก redirect ไป `/dashboard` แบบไม่มีเงื่อนไข มาเป็นเรียก `auth()` ก่อน แล้ว redirect เฉพาะ session ที่มีอยู่จริง (ไป `/dashboard` หรือ `/onboarding` ตาม `profileCompleted`) — ถ้าไม่มี session จะ render component `SongsueLanding` แทนการ redirect
- **`src/components/home/SongsueLanding.tsx`** (ไฟล์ใหม่ ~480 บรรทัด) — หน้า landing แบบ client component เดียว ใช้ `framer-motion` (dependency ใหม่ใน `package.json`/`package-lock.json`) ทำ scroll-linked animation (fade/scale ของ hero, sticky reveal section, staggered text) พร้อม media query เช็ค `prefers-reduced-motion` เพื่อ fallback เป็นอนิเมชันแบบง่าย
  - Countdown logic: เทียบเวลาปัจจุบันกับ registration-open timestamp คงที่ (24 ก.ค. 69 00:00 เวลาไทย/Bangkok) — ก่อนถึงเวลาแสดง countdown + ปุ่ม disabled, หลังถึงเวลาแสดงปุ่ม sign-in ที่เรียก `signIn` ของ next-auth ตรง ๆ (auth flow เดิมของระบบ ไม่มี endpoint ใหม่)
- **`src/components/home/songsue-copy.ts`** (ไฟล์ใหม่) — เก็บ copy เล่าเรื่องสองภาษา (TH/EN) แยกจาก `src/lib/i18n.ts` โดยตั้งใจ เพราะเป็น narrative/marketing copy เฉพาะหน้านี้ ไม่ใช่ UI string ของแอปหลัก — ผู้ใช้ mm/cn จะเห็น fallback เป็นเวอร์ชันภาษาอังกฤษเฉพาะหน้านี้เท่านั้น ไม่กระทบ i18n ของแอปส่วนอื่น
- **Chore ติดมากับ branch เดียวกัน**: ปรับ `.claude/agents/drizzle-migration-author.md` ให้ใช้ model tier `sonnet` (จากเดิม `opus`) — ไฟล์ agent อีก 3 ไฟล์ที่เคยตั้งใจจะปรับ พบว่ามีคน merge เปลี่ยนเข้า main ไปแล้วก่อนหน้านี้ จึงเหลือไฟล์นี้ไฟล์เดียวที่เป็นการเปลี่ยนแปลงใหม่จริง ๆ

### DB / Migration

- **ไม่มี DB migration ใหม่** — เป็น UI/routing layer ล้วน ไม่มีการเพิ่ม/แก้ตาราง ไม่ต้องรัน `db:migrate:container`

### PDPA / สิทธิ์เข้าถึง

- **ไม่มีการเปลี่ยน access control หรือ PDPA gating** — หน้า landing เป็น public route ที่ไม่ดึงข้อมูลผู้ใช้ใด ๆ; ผู้ที่มี session อยู่แล้วยังคงถูก gate ด้วย logic เดิม (`auth()` + `profileCompleted` check) เหมือนก่อนเปลี่ยน; ปุ่มลงทะเบียนเรียก next-auth `signIn` มาตรฐานเดิม (จำกัด `@cmu.ac.th` เหมือนเดิม) ไม่มี API หรือ read path ใหม่สำหรับข้อมูลการแพทย์
