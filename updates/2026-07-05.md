  - **ปุ่มลงทะเบียนจะเปิดใช้งานอัตโนมัติเมื่อถึงเวลา** — ก่อนวันที่ 24 ก.ค. 69 เวลา 00:00 น. หน้าเว็บจะแสดงตัวนับถอยหลัง (วัน/ชั่วโมง/นาที/วินาที) แบบเรียลไทม์พร้อมปุ่มที่ยังกดไม่ได้ พอถึงเวลาที่กำหนดปุ่มจะเปลี่ยนเป็น "Register with Google" ให้กดล็อกอินเข้าระบบได้ทันที (จำกัดเฉพาะอีเมล @cmu.ac.th เหมือนระบบส่วนอื่นของเว็บ)

  - รองรับผู้ใช้ที่ตั้งค่าอุปกรณ์ให้ลดทอนการเคลื่อนไหว (reduce motion) โดยจะแสดงผลแบบ fade ธรรมดาไม่มีอนิเมชันซับซ้อนให้

## ฝั่งทีม (technical changelog)

### Routing / Landing Page

- **`src/app/page.tsx`** — เปลี่ยนจาก redirect ไป `/dashboard` แบบไม่มีเงื่อนไข มาเป็นเรียก `auth()` ก่อน แล้ว redirect เฉพาะ session ที่มีอยู่จริง (ไป `/dashboard` หรือ `/onboarding` ตาม `profileCompleted`) — ถ้าไม่มี session จะ render component `SongsueLanding` แทนการ redirect
- **`src/components/home/SongsueLanding.tsx`** (ไฟล์ใหม่ ~480 บรรทัด) — หน้า landing แบบ client component เดียว ใช้ `framer-motion` (dependency ใหม่ใน `package.json`/`package-lock.json`) ทำ scroll-linked animation (fade/scale ของ hero, sticky reveal section, staggered text) พร้อม media query เช็ค `prefers-reduced-motion` เพื่อ fallback เป็นอนิเมชันแบบง่าย
  - Countdown logic: เทียบเวลาปัจจุบันกับ registration-open timestamp คงที่ (24 ก.ค. 69 00:00 เวลาไทย/Bangkok) — ก่อนถึงเวลาแสดง countdown + ปุ่ม disabled, หลังถึงเวลาแสดงปุ่ม sign-in ที่เรียก `signIn` ของ next-auth ตรง ๆ (auth flow เดิมของระบบ ไม่มี endpoint ใหม่)
- **`src/components/home/songsue-copy.ts`** (ไฟล์ใหม่) — เก็บ copy เล่าเรื่องสองภาษา (TH/EN) แยกจาก `src/lib/i18n.ts` โดยตั้งใจ เพราะเป็น narrative/marketing copy เฉพาะหน้านี้ ไม่ใช่ UI string ของแอปหลัก — ผู้ใช้ mm/cn จะเห็น fallback เป็นเวอร์ชันภาษาอังกฤษเฉพาะหน้านี้เท่านั้น ไม่กระทบ i18n ของแอปส่วนอื่น
- **Chore ติดมากับ branch เดียวกัน**: ปรับ `.claude/agents/drizzle-migration-author.md` ให้ใช้ model tier `sonnet` (จากเดิม `opus`) — ไฟล์ agent อีก 3 ไฟล์ที่เคยตั้งใจจะปรับ พบว่ามีคน merge เปลี่ยนเข้า main ไปแล้วก่อนหน้านี้ จึงเหลือไฟล์นี้ไฟล์เดียวที่เป็นการเปลี่ยนแปลงใหม่จริง ๆ

### DB / Migration

- **ไม่มี DB migration ใหม่** — เป็น UI/routing layer ล้วน ไม่มีการเพิ่ม/แก้ตาราง ไม่ต้องรัน `db:migrate:container`

### PDPA / สิทธิ์เข้าถึง

- **ไม่มีการเปลี่ยน access control หรือ PDPA gating** — หน้า landing เป็น public route ที่ไม่ดึงข้อมูลผู้ใช้ใด ๆ; ผู้ที่มี session อยู่แล้วยังคงถูก gate ด้วย logic เดิม (`auth()` + `profileCompleted` check) เหมือนก่อนเปลี่ยน; ปุ่มลงทะเบียนเรียก next-auth `signIn` มาตรฐานเดิม (จำกัด `@cmu.ac.th` เหมือนเดิม) ไม่มี API หรือ read path ใหม่สำหรับข้อมูลการแพทย์

## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น) — อัปเดตเพิ่ม

- **หน้า Login (`/login`) เปลี่ยนมาใช้หน้า Landing แบบใหม่แล้ว** เดิมหน้านี้ยังเป็นดีไซน์เก่า (ไม่มีการแจ้งเตือนเมื่อ login ไม่สำเร็จ) ตอนนี้เปลี่ยนมาใช้หน้า cinematic scroll เดียวกับหน้าแรกของเว็บ
  - คนที่เคยลงทะเบียน/มีบัญชีอยู่แล้วและกลับมา login ใหม่ จะเห็นปุ่ม "Sign in with Google" ที่กดได้ทันทีเสมอ ไม่ต้องรอถึงวันเปิดลงทะเบียน (24 ก.ค. 69) เหมือนหน้าแรก — พร้อมข้อความต้อนรับกลับแทนข้อความชวนลงทะเบียน
  - ถ้า login ไม่สำเร็จ (เช่นเปิดลิงก์จากแอปในตัวของ LINE/Gmail/Instagram แล้ว Google OAuth ล้มเหลว) จะมีกล่องแจ้งเตือนสีแดงอธิบายว่า sign-in ไม่สำเร็จ พร้อมคำแนะนำให้เปิดลิงก์ในเบราว์เซอร์จริงแทน
  - หน้า Landing เก่า (`LandingUI`) ที่ไม่ได้ใช้แล้วถูกลบทิ้งไปเรียบร้อย

## ฝั่งทีม (technical changelog) — อัปเดตเพิ่ม

### Routing / Landing Page

- **`src/components/home/SongsueLanding.tsx`** เพิ่ม prop `variant?: "home" | "login"` (default `"home"`) และ `authError?: string | null`:
  - `variant === "login"`: ปุ่ม sign-in แสดงและกดได้เสมอ ไม่ผูกกับ `countdown.isOpen` เหมือน variant `"home"`; สลับ copy ส่วน CTA เป็นข้อความต้อนรับผู้ใช้เดิม (`t.welcome`, `t.accessDashboard`, `t.signInBtn` จาก i18n) แทน narrative copy ของ `songsue-copy.ts`
  - เพิ่มกล่อง alert (`role="alert"`) แสดงเมื่อมี `authError` — ใช้ copy key `signInErrorTitle` / `signInErrorBody` / `signInErrorHint` จาก i18n เดิม (ของ error banner เก่าที่เคยอยู่ใน `LandingUI`)
  - เอา `copy.cta.footnote` (ข้อความ "ใช้บัญชี Google ของ CMU") ออกจากหน้า CTA เพราะซ้ำกับข้อความบนปุ่ม/หน้า login แล้ว — key `footnote` ถูกลบออกจาก type `SongsueCopy` และทั้งสองภาษาใน `songsue-copy.ts`
- **`src/app/login/page.tsx`** เปลี่ยนจาก `<LandingUI userCount={31} authError={...} enableDevLogin={...} />` มาเป็น `<SongsueLanding variant="login" authError={error ?? null} />` — logic การอ่าน `?error=` จาก `searchParams` และการ redirect ผู้ใช้ที่ login แล้วไป dashboard/onboarding เหมือนเดิมทุกประการ ไม่ได้แตะ
- **ลบไฟล์ `src/components/home/LandingUI.tsx`** (352 บรรทัด) — เป็น component เก่าที่ไม่มีที่ไหนเรียกใช้แล้วหลังเปลี่ยนหน้า login
- **แก้ก่อน merge**: ระหว่างทดสอบ local เคยแก้วันที่เปิดลงทะเบียนใน `SongsueLanding.tsx` เป็น `2020-01-01` ชั่วคราวเพื่อทดสอบสถานะ "เปิดแล้ว" — ตรวจพบก่อน commit และคืนค่ากลับเป็น `2026-07-24` ที่ถูกต้องก่อน push/merge

### DB / Migration

- ไม่มี — เป็นการแก้ routing/UI component ล้วน ไม่มีตารางหรือ migration ใหม่

### PDPA / สิทธิ์เข้าถึง

- ไม่มีการเปลี่ยน access control หรือ auth flow จริง — ยังเรียก `signIn("google", ...)` ของ next-auth เหมือนเดิม (จำกัด `@cmu.ac.th` เหมือนเดิม); การเปลี่ยนแปลงอยู่ที่ UI/copy ของหน้า login เท่านั้น
