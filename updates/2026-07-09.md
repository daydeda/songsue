## ฝั่งทีม (technical changelog)

### แก้ header ของ modal Attendance บนมือถือดันรายชื่อล้นจอ (PR #162)

- **ปัญหาเดิม:** modal "Attendance" (เปิดจากปุ่มบนการ์ดกิจกรรมในหน้า `/admin/events`) มีปุ่ม "Export Excel" กับ "Strike No-shows" วางอยู่ในแถบ header ที่เป็น sticky ร่วมกับชื่อกิจกรรม/จำนวนเช็คอิน — บนมือถือแถบ header จึงสูงมาก ดันให้รายชื่อ (roster) ที่เป็นเนื้อหาหลักตกไปอยู่นอกจอ ต้องเลื่อนหาถึงจะเห็น
- **`src/app/admin/events/page.tsx`** — ย้ายปุ่ม "Export Excel" และ "Strike No-shows" ออกจาก `.attendance-modal-header` (sticky) ไปไว้ในแถบใหม่ `.attendance-modal-actions-bar` (ไม่ sticky) ที่อยู่ถัดลงมา เลื่อนหายไปพร้อมเนื้อหาส่วนอื่นเหมือนแถบ filter เดิม — header ที่ sticky อยู่ตอนนี้เหลือแค่ชื่อกิจกรรม + จำนวนเช็คอิน + ปุ่มปิด (X)
- เพิ่ม CSS class ใหม่ 2 ตัว: **`.attendance-modal-close-btn`** (ปุ่มปิด วงกลม 40px บนมือถือ / 48px บน desktop) และ **`.attendance-action-btn`** (สไตล์ร่วมของปุ่ม Export/Strike ทั้งคู่ — สูง 40px, `flex: 1` วางเรียงกันเต็มความกว้างบนมือถือ แล้วกลับไปเป็นปุ่ม pill ขนาดเต็ม 48px แบบเดิมที่ breakpoint desktop เดิม 1024px — หน้าตาบน desktop ไม่เปลี่ยนเลย)
- รวบโค้ดที่เรียก `events.find(e => e.id === activeEventId)` ซ้ำหลายจุดให้เหลือตัวแปรเดียว `activeEvent`, และ precompute `showExportButton`/`showStrikeButton` ไว้ล่วงหน้า — ตรรกะการซ่อน/แสดงปุ่ม (`canExportAttendance`/`canSeeExportButton`/`canApplyStrikes`) เหมือนเดิมทุกประการ **ไม่มีสิทธิ์เข้าถึงส่วนไหนเปลี่ยน** เป็นแค่การจัดวาง layout/CSS
- ตรวจสอบผ่าน dev-bypass login กับ local DB ด้วย Playwright screenshot ที่ viewport 390px (มือถือ) และ 1400px (desktop) — มือถือเห็น header บางลงและ roster โผล่ในจอทันที ส่วน desktop หน้าตาปุ่มเหมือนเดิมทุกจุด
