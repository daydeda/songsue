## ฝั่งทีม (technical changelog)

### ฟีเจอร์ใหม่: Event Staff picker สำหรับข้อเสนอกิจกรรมของสาขา (PR #186)

- **root cause / gap ที่เจอ:** `staffUserIds` มีอยู่แล้วเป็น column กลางบน `event_proposals` (ใช้ร่วมกับฝั่งชมรม) แต่ `MajorProposeEventSection.tsx` ไม่เคยมี UI ให้เลือกเลย เพราะสาขาไม่มีตาราง roster แบบ `club_members` ให้ query รายชื่อสมาชิกมาแสดง
- **`src/modules/majors/majors.service.ts`** (ใหม่) — `MajorsService.getMajorMembers(code)` เทียบเท่า `ClubsService.getClubMembers` ของฝั่งชมรม โดย query ตรงจาก `users.major` แทนที่จะมี junction table แยก
- **`GET /api/admin/majors/[code]/members`** (ใหม่) — คืนรายชื่อ (name, studentId) ของนักศึกษาในสาขานั้น จำกัดสิทธิ์เฉพาะ `super_admin`/`admin` หรือ major_president ของสาขาตัวเองเท่านั้น (ตรวจสอบฝั่ง server ผ่าน `EventScopeService.getPresidentScope` ไม่เชื่อ `:code` ที่ client ส่งมาตรง ๆ) — mirror gate pattern เดียวกับ `GET /api/admin/clubs/[id]/members`
- **`POST /api/admin/event-proposals`** และ **`PATCH /api/admin/event-proposals/[id]` (resubmit)** เพิ่ม branch ตรวจสอบ `staffUserIds` ของข้อเสนอฝั่งสาขาด้วย — กรองทิ้งทุก id ที่ไม่ใช่นักศึกษาในสาขานั้นจริงก่อนบันทึกเสมอ (ไม่เชื่อ id ที่ client ส่งมา) เหมือน pattern เดิมที่มีอยู่แล้วฝั่งชมรม
- **`MajorProposeEventSection.tsx`:** เพิ่ม UI เลือก/ค้นหา/ถอดทีมงาน, ป้ายจำนวนทีมงานที่เสนอบนการ์ดรายการและหน้ารายละเอียด — mirror `ProposeEventSection.tsx` (ฝั่งชมรม) ทุกจุด ต่างแค่แหล่งข้อมูลที่ดึงมา
- หน้า "Create Event from Proposal" (`admin/events/page.tsx`) ที่ prefill `staffUserIds` จาก proposal ไว้อยู่แล้วไม่ต้องแก้อะไรเพิ่ม เพราะอ่านค่าแบบ generic ไม่แยก clubId/majorCode
- ตรวจสอบด้วย `npx tsc --noEmit`, `npm run lint`, `npm run build` (ผ่านทั้งหมด ไม่มี error ใหม่จากแพ็กเกจนี้ — error เดิมใน `battle-flow.test.ts` ไม่เกี่ยวข้องกับแพ็กเกจนี้)
- ไม่มีการเปลี่ยน schema/migration — `staffUserIds` เป็น column ที่มีอยู่แล้วบน `event_proposals`
- ตัดแพ็กเกจนี้เป็น **tag/release v1.6.0**
