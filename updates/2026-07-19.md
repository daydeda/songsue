### Sync กับ upstream smocamt-website (PR #5)

- Merge 128 commits จาก `daydeda/smocamt-website` main เข้า songsue โดยคงฟีเจอร์เฉพาะของ songsue ไว้ทั้งหมด (cinematic landing page, 4-faculty houses, การจัดบ้านตามเวลาเช็คอิน) — ได้ฟีเจอร์ฝั่ง upstream ที่ตามมาด้วย เช่น การแยกสิทธิ์ตำแหน่งงานตามชมรม/สาขา/SMO/ANUSMO, การแก้ PDPA/access-control หลายจุด, งานฝั่ง houses และการแก้ CI/test fixtures
- ยืนยันด้วย `git merge-tree` ก่อน push ว่าไม่มี conflict จริงระหว่างสาขานี้กับ `origin/main` ของ songsue
- **ยืนยันขอบเขต:** PR นี้ผูกกับ `daydeda/songsue` (remote `origin`) เท่านั้น ไม่มีการ push อะไรไปยัง `daydeda/smocamt-website` (remote `upstream`) หรือแตะ production deploy จริง (`activecamt.camt.cmu.ac.th`) เลย — `.github/workflows/docker-publish.yml` ของ repo นี้ publish ไปที่ `ghcr.io/daydeda/songsue` (แยกจาก tag ของ production) และไม่มี Portainer auto-redeploy step มาตั้งแต่ PR #4 แล้ว
- **DoorCastle3D:** ตัด wall mesh สีดำทึบและ stone diffuse texture ที่ทำให้ arch out สีน้ำตาลผิดโทน (ต้นฉบับเป็นภาพอิฐแดง ไม่ใช่หินเทา) ปรับแสง (ลด ambient, เพิ่ม key light สีอุ่น + rim light สีเย็น) และเปลี่ยนจาก HDRI "night" สีน้ำเงินทั่วไปเป็น Lightformer โทนส้ม/ทองที่ match กับสีของหน้าเว็บ
- **FlagFlutter3D:** บังคับ re-measure ขนาด canvas หนึ่งเฟรมหลัง mount เพื่อแก้บั๊กที่ react-three-fiber's Canvas re-measure ขนาดแค่ตอน window resize/scroll เท่านั้น ทำให้ธงเรนเดอร์เล็กผิดขนาดจนกว่า user จะเลื่อนหน้าจอ
- **SongsueLanding:** เปลี่ยน section เต็มความสูงจาก `vh` เป็น `svh` (กัน mobile browser chrome ทำให้ viewport กระโดดตอนเลื่อน), เพิ่ม `goToHouse` handler ให้ pagination dots กดได้ (พร้อม `aria-label`/`aria-current` สำหรับ accessibility), ปรับ drag-to-swipe threshold ของ carousel
- ตรวจสอบด้วย `npm run build` (ผ่าน) และ `npm test` (111/111 ผ่าน)
- Bump `package.json` เป็น 1.9.0 (ครั้งแรกที่ songsue ตัด tag/release ของตัวเอง แยกจาก tag ของ upstream)
- ตัดแพ็กเกจนี้เป็น **tag/release v1.9.0**

### เตรียม deploy songsue ขึ้น production แยกจาก activecamt (PR #7)

- **`docker-stack.songsue.yml`** (ไฟล์ใหม่) — Portainer stack แยกอิสระสำหรับ `songsue.camt.cmu.ac.th` (image `ghcr.io/daydeda/songsue`, DB/volumes ของตัวเอง, port `10781` — พอร์ตว่างตัวถัดไปในช่วง `10700-10799` ที่ IT ของ CAMT อนุมัติให้โปรเจกต์นี้ใช้ ถัดจาก `10780` ที่ activecamt ใช้อยู่แล้ว) — ไม่แชร์อะไรกับ stack ของ activecamt เลยนอกจากเครื่อง dev2 เดียวกัน
- **`docs/songsue-deploy.md`** (ไฟล์ใหม่) — runbook deploy ฉบับย่อ อ้างอิงขั้นตอนจริงจากเอกสาร dev2 Portainer ของ IT — ไม่มีขั้นตอน migrate ข้อมูล/ไฟล์เพราะเป็นฐานข้อมูลใหม่ล้วน ๆ
- **OAuth:** ใช้ Google OAuth client ตัวเดียวกับ activecamt ได้เลย (เพิ่มแค่ redirect URI ใหม่ในฝั่ง Google Cloud Console) — โค้ดฝั่ง `src/auth.ts` ไม่มีการจำกัดโดเมน `@cmu.ac.th` อยู่แล้วในปัจจุบัน จึงไม่ต้องแก้โค้ดเพื่อให้ songsue เปิดรับทุกบัญชี Google
- **ลบไฟล์ที่เป็นของ activecamt production จริง:** `docker-stack.yml` และ `docs/supabase-to-university-server-cutover.md` (มีพอร์ต/โดเมน/ชื่อ image ของระบบ activecamt.camt.cmu.ac.th ที่ใช้งานจริงอยู่) ถูกดึงเข้ามาใน repo นี้ตอน merge upstream ก่อนหน้านี้เท่านั้น ตอนนี้ repo `daydeda/songsue` เปิดเป็น public แล้ว การเก็บรายละเอียด infra production จริงของอีกทีมไว้จึงไม่มีประโยชน์และเสี่ยงเปิดเผยเกินจำเป็น — ลบทิ้งไม่กระทบ deploy จริงของ activecamt เลยเพราะ repo `daydeda/smocamt-website` มีไฟล์ของตัวเองแยกต่างหากอยู่แล้ว
- ยังเหลือขั้นตอนที่ต้องให้ CAMT IT ทำ: ผูก reverse proxy ของ `songsue.camt.cmu.ac.th` เข้ากับพอร์ต `10781`
- ตรวจสอบด้วย `npm run lint` (error/warning เดิมทั้งหมดอยู่ในไฟล์ที่ไม่เกี่ยวกับแพ็กเกจนี้)
