### แก้บั๊กจากการ deploy จริง + เตรียม super-admin/auto-deploy (PR #9–#12)

พบและแก้ทีละจุดระหว่างทดลอง deploy จริงบน dev2 Portainer:

- **แก้ข้อความ runbook เรื่อง auto-deploy เกินจริง (PR #9)** — เดิมเขียนว่าเปิด GitOps webhook แล้วจะ auto-deploy ได้เลย แต่ตรวจพบจริงจาก screenshot ว่าปุ่ม "Re-pull image"/"Force redeployment" บน dev2 ถูกล็อกไว้เป็นฟีเจอร์ของ Portainer Business Edition (เครื่องนี้เป็น Community Edition) — แก้เอกสารให้บอกวิธีที่ยืนยันแล้วว่าใช้ได้จริงบน free tier แทน (Portainer → Update the stack → "Pull latest image version")
- **แก้ error `relation "users" does not exist` (PR #10)** — `db:migrate:container` เป็น incremental migration เท่านั้น (สมมติว่ามีตารางพื้นฐานอยู่แล้ว) ใช้กับฐานข้อมูลใหม่เอี่ยมไม่ได้ทันที ต้องรัน `npm run db:push` ก่อนเพื่อสร้าง schema เต็มจาก `src/db/schema.ts` แล้วค่อยรัน `db:migrate:container` ตาม — แก้ runbook Step 4 ให้รันทั้งสองคำสั่งตามลำดับ
- **แก้ error `drizzle.config.json file does not exist` (PR #11)** — สาเหตุจริงคือ Dockerfile stage `runner` ไม่เคย copy ไฟล์ `drizzle.config.ts` เข้า production image เลย (มีแค่ `src`, `scripts`, `tsconfig.json`) ทำให้ `db:push` (ซึ่งต้องใช้ config นี้) รันไม่ได้ตั้งแต่แรก — เพิ่ม `COPY --from=builder /app/drizzle.config.ts` เข้า Dockerfile
- **เพิ่มช่องทางตั้งค่า `super_admin` และ auto-deploy จริง (PR #12):**
  - `SUPER_ADMIN_EMAILS` (comma-separated) ตอนนี้ผูกเข้ากับ `docker-stack.songsue.yml` แล้วจริง ๆ (เดิมมีแค่คำอธิบายในเอกสารแต่ไม่เคยส่งค่าเข้า container) — ตั้งอีเมลแล้ว sign-in ใหม่ ระบบจะ promote เป็น `super_admin` อัตโนมัติ (คนละกลไกกับ `elevate-admin.ts` ที่ให้ได้แค่ `admin`)
  - เพิ่มขั้นตอน "Trigger Portainer redeploy" ใน `docker-publish.yml` ตามแบบที่ `smocamt-website` ใช้งานจริงใน production แต่แยก secret (`SONGSUE_PORTAINER_WEBHOOK_URL`) เป็นคนละตัวกับของ activecamt เด็ดขาด เพื่อไม่ให้ CI ของ songsue แตะ stack จริงของ activecamt ได้ — ยังไม่ยืนยัน 100% ว่า webhook นี้บังคับ pull image ใหม่จริงบน dev2 (ต้องรอทดสอบกับการ push จริงครั้งถัดไป) เพราะติดข้อจำกัด Business Edition ข้างต้น จึงยังต้องพึ่ง manual pull เป็น fallback ที่มั่นใจได้กว่า
- ตรวจสอบด้วย `npm run lint` (error/warning เดิมทั้งหมดอยู่ในไฟล์ที่ไม่เกี่ยวข้องกับการเปลี่ยนแปลงนี้)
- ตัดแพ็กเกจนี้เป็น **tag/release v1.9.2**
