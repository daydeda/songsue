- **เล่น P2P Battle ได้ทุกคนแล้ว** — เดิมเปิดให้เฉพาะ SMO/ANUSMO/แอดมิน ทดสอบบนระบบจริงก่อน ตอนนี้เปิดให้นักศึกษาทุกคนที่ล็อกอินเข้าเล่นได้แล้ว

## ฝั่งทีม (technical changelog)

### ระบบ sync กิจกรรม/การลงทะเบียนจาก ActiveCAMT + เปิด P2P Battle เต็มรูปแบบ (PR #26)

- **Sync ทิศทางเดียวจาก ActiveCAMT → Songsue** — เมื่อแอดมิน ActiveCAMT ตั้งค่ากิจกรรมให้ "นับแต้มให้ Songsue ด้วย" และนักศึกษาลงทะเบียน/เช็คอิน QR ที่ฝั่ง ActiveCAMT ระบบใหม่ (`src/modules/integrations/activecamt-sync.service.ts` + `/api/integrations/activecamt/events`, `/register`) จะ mirror ข้อมูลกิจกรรมและสถานะการเข้าร่วมมาที่ตาราง `events`/`attendance` ของ Songsue ให้ cron แจกแต้มบ้าน (`checkAndAwardPastEventPoints`) เห็นและนับแต้มให้ถูกบ้าน — ActiveCAMT ยังคงเป็นเจ้าของการลงทะเบียน/โควตา/เช็คอิน QR เต็มรูปแบบเหมือนเดิม ฝั่ง Songsue ปิดกั้นการลงทะเบียน/ยกเลิก/สแกนสำหรับกิจกรรมที่ถูก mirror มาโดยเช็คคอลัมน์ใหม่ `events.externalSource` ในทุกจุด (register/cancel API, scan API, หน้าสแกนแอดมิน, การ์ดกิจกรรมบน dashboard)
- **บัญชีที่ sync มาเป็นแบบ PDPA-minimal** — สร้างบัญชี Songsue จริงให้นักศึกษาโดยอัตโนมัติที่ email เดียวกัน (ใช้ email เป็น join key ข้ามสองระบบ) แต่ `profileCompleted`/`pdpaConsent` เป็น false เสมอ และไม่แตะข้อมูลการแพทย์ใด ๆ เลย — ต้องให้นักศึกษายืนยัน PDPA consent ของ Songsue เองถ้าเข้าระบบตรง ๆ ในภายหลัง ส่วนบ้านจะถูกจัดให้แบบ balanced ตามระบบบ้านของ Songsue เอง (ไม่รับค่าบ้านจาก ActiveCAMT มาโดยตรง เพราะสองระบบใช้ house model คนละแบบ) และจัดให้ก็ต่อเมื่อ sync มาถึงสถานะ "attended" ครั้งแรกเท่านั้น ไม่ใช่ตอน "registered"
- **DB migration (step 88):** เพิ่มคอลัมน์ `events.external_source`/`events.external_id` + unique partial index `events_external_source_id_unique` (`WHERE external_id IS NOT NULL`) เป็น additive/idempotent ล้วน รันผ่าน `/safe-deploy` บน Supabase prod แล้วก่อน merge โค้ดที่อ่าน/เขียนคอลัมน์นี้
- **รีวิว PDPA/access-control** ผ่าน agent `pdpa-access-guard` ก่อน merge — ตรวจครบ: บัญชี sync ไม่แตะข้อมูลการแพทย์, audit log เขียนใน transaction เดียวกับการเปลี่ยนแปลงข้อมูลเสมอ, endpoint ใหม่ทั้งสองเช็ค bearer secret (`ACTIVECAMT_SYNC_SECRET`, constant-time, fail-closed) ก่อนแตะ DB ทุกครั้ง, และจุดกันการลงทะเบียน/สแกนซ้ำครอบคลุมทุก path จริง ไม่มี finding ระดับ crucial/moderate (มีแค่ 2 จุด low-severity เรื่อง reliability ที่ไม่กระทบความปลอดภัย)
- **เปิด P2P Battle ให้ทุก role ที่ล็อกอินแล้ว** — เดิมจำกัดไว้เฉพาะ SMO/ANUSMO/แอดมินสำหรับทดสอบบนระบบจริง (`src/lib/battle-access.ts`, `src/proxy.ts`, `src/app/battle/layout.tsx`) ตอนนี้เปิดให้ทุกคนแล้ว
- **แก้บั๊ก `GET /api/profile`** — เดิมถ้า session ยังใช้ได้แต่แถว user ในตารางถูกลบไปแล้ว (เช่นหลัง reset/reseed DB) จะโยน error "Value is not JSON serializable" แทนที่จะตอบ error ที่อ่านง่าย ตอนนี้ตอบ 404 ตรง ๆ แทน
- ตรวจสอบด้วย `npm run build` (ผ่าน, มี route ใหม่ทั้งสองขึ้นในผลลัพธ์), `npm run lint` (ไม่มี error/warning ใหม่ — error เดิมทั้งหมดยังอยู่ในไฟล์ที่ไม่เกี่ยวข้อง เช่น DoorCastle3D.tsx/SoftShadows.tsx), และ `npm test` (111/111 ผ่าน)
- **ยังไม่เสร็จ:** ฝั่ง ActiveCAMT (repo `smocamt-website` แยกต่างหาก) ที่ยิง sync มาหา endpoint พวกนี้ยังไม่ได้ทำ — ต้องตั้ง `ACTIVECAMT_SYNC_SECRET` ให้ตรงกันทั้งสองฝั่งบน Vercel ก่อนใช้งานจริงได้
