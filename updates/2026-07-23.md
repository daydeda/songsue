## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **แก้บั๊กรูปแบนเนอร์และทีมงานของกิจกรรมที่เชื่อมกับ ActiveCAMT ไม่ sync มา Songsue** — เดิมถ้าแอดมินแก้ไขกิจกรรมฝั่ง ActiveCAMT (เช่น เพิ่มรูปแบนเนอร์ หรือมอบหมายทีมงานประจำกิจกรรม) แล้วกด "Resync to Songsue" ข้อมูลสองส่วนนี้จะไม่ถูกส่งมาเลย ตอนนี้แก้ให้ sync มาถูกต้องแล้วทุกครั้งที่สร้าง/แก้ไข/resync กิจกรรม
- **หน้าแรกมีเพลงประจำบ้านแล้ว** — วิดีโอพื้นหลังของแต่ละบ้านในหน้าเลือกบ้าน (flag carousel) มีเสียงเพลงประกอบให้ฟังได้ (เดิมมีแต่ภาพ ไม่มีเสียง) เปิดเสียงให้อัตโนมัติเมื่อเข้าหน้า พร้อมปุ่มเปิด/ปิดเสียงมุมขวาบนถ้าไม่อยากฟัง และการ์ดบ้านจะเลื่อนเปลี่ยนเองทุก 15 วินาทีถ้าไม่ได้กดเลือกเอง (ปิดการเลื่อนอัตโนมัติให้เองถ้าเครื่องตั้งค่า "ลดการเคลื่อนไหว" ไว้)
- **ลงทะเบียนได้แล้ววันนี้** — ข้อความหน้าแรกเปลี่ยนจากนับถอยหลังเป็นเชิญชวนลงทะเบียนด้วย Google ตรง ๆ เพราะระบบเปิดลงทะเบียนแล้ว (23 ก.ค. 69)

## ฝั่งทีม (technical changelog)

### แก้ payload sync กิจกรรมให้พก imageUrl/imageUrls และ staffUserIds ไปด้วย (PR #31)

- **สาเหตุ:** `syncEventToSongsue` (ฝั่ง ActiveCAMT, repo `smocamt-website`) และ endpoint รับ sync (`/api/integrations/activecamt/events` ฝั่งนี้) ไม่เคยรับ-ส่งฟิลด์แบนเนอร์ (`imageUrl`/`imageUrls`) หรือทีมงานประจำกิจกรรม (`staffUserIds`) เลยตั้งแต่เริ่มมี sync — ทั้งตอนสร้าง/แก้ไขกิจกรรม และตอนกดปุ่ม resync ด้วยมือ
- **ฝั่ง ActiveCAMT (`smocamt-website`, branch `fix/songsue-event-sync-banner-staff`):** ทั้ง 3 จุดที่ยิง sync (สร้างกิจกรรม, แก้ไขกิจกรรม, resync ด้วยมือ) ตอนนี้ resolve `staffUserIds` (user id ภายในของ ActiveCAMT) เป็น email/name ก่อนส่ง เพราะ user id ฝั่ง ActiveCAMT ไม่มีความหมายฝั่ง Songsue เลย พร้อมส่งค่าแบนเนอร์ปัจจุบันไปด้วย
- **ฝั่ง Songsue (repo นี้, PR #31):** `ActiveCamtSyncService.upsertExternalEvent` เขียน `imageUrl`/`imageUrls` ลงกิจกรรมโดยตรง และ resolve ทีมงานแต่ละคนด้วย email เป็น join key เหมือน sync การลงทะเบียนที่มีอยู่แล้ว — ถ้ายังไม่มีบัญชี Songsue ที่ email นั้นจะสร้างบัญชีขั้นต่ำให้ (ไม่มี `pdpaConsent`, ไม่มีข้อมูลการแพทย์ใด ๆ) แล้วเขียน id ที่ resolve ได้ลง `events.staffUserIds`
- ทั้งสอง payload ส่งค่า "ปัจจุบันทั้งหมด" ไม่ใช่ partial patch — เอาทีมงานออกจาก ActiveCAMT แล้ว resync ก็จะเอาออกจาก Songsue ด้วยเช่นกัน
- ตรวจสอบด้วย `npx tsc --noEmit` (ผ่านทั้งสอง repo), `npm run lint` (ไม่มี error/warning ใหม่ในไฟล์ที่แก้ทั้งสอง repo), `npm test` (111/111 ผ่านทั้งสอง repo) — ยังไม่ได้ทดสอบ E2E จริงกับ local DB เพราะ Docker ไม่ได้เปิดอยู่ตอนแก้ (แนะนำให้ทดสอบ resync จริงอีกครั้งหลัง deploy ทั้งสองฝั่งแล้ว)
- ตัดแพ็กเกจนี้เป็น **tag/release v1.16.1**

### เพลงประจำบ้าน + carousel เลื่อนอัตโนมัติ + เปิดลงทะเบียน (PR #33)

- **`HouseVideo` แยกเป็น component ของตัวเอง** (`src/components/home/SongsueLanding.tsx`) — ให้วิดีโอที่กำลังจะออกกับที่กำลังจะเข้าเป็นคนละ DOM node จริง ๆ ระหว่าง crossfade เพื่อให้ปรับ volume ตาม opacity ของแต่ละตัวได้อิสระ ไม่แย่งกันคุม ref เดียว — เสียงจึงเฟดเข้า/ออกพร้อมภาพ ไม่ใช่ตัดขาดทันทีตอนเปลี่ยนบ้าน
- **ปุ่มเปิด/ปิดเสียง** เริ่มต้นเปิดเสียงไว้ (volume 0.35) แต่ swallow rejection จาก autoplay-with-sound policy ของเบราว์เซอร์อัตโนมัติ (fallback เป็น mute แทนที่จะเงียบค้าง) และซ่อนปุ่มถ้าเครื่องตั้ง prefers-reduced-motion ไว้
- **Carousel เลื่อนอัตโนมัติทุก 15 วินาที** ผูก effect ไว้กับ `activeIndex` (ไม่ใช่ interval คงที่) เพื่อให้การเลื่อนด้วยมือ (จุด/ลาก/ปุ่มลูกศร) ดันรอบถัดไปออกไปแทนที่จะชนกัน และข้ามการเลื่อนอัตโนมัติทั้งหมดถ้า prefers-reduced-motion (WCAG 2.2.2)
- **`DoorCastle3D`** เพิ่ม prop `onOpenChange` ยิงทันทีตอนคลิกประตู (ก่อนแอนิเมชัน 3D เปิด/ซูมจะจบ) ให้พื้นหลัง DOM ด้านหลัง canvas โปร่งใสเริ่มทรานซิชันซูมพร้อมกันได้เลย แทนที่จะค้างนิ่งจนกว่ากล้อง 3D จะบินผ่านซุ้มประตูเสร็จ
- **อัปเดตข้อความ CTA** ทั้ง TH/EN ให้ตรงกับสถานะเปิดลงทะเบียนจริง (`src/components/home/songsue-copy.ts`)
- ตรวจสอบด้วย `npm run build` (ผ่าน) และ `npm run lint` (ไม่มี error/warning ใหม่ — error เดิม 4 จุดใน `DoorCastle3D.tsx` เป็นโค้ดเก่าตั้งแต่ 18 ก.ค. ยืนยันด้วย `git blame`, ไม่เกี่ยวกับการแก้ไขนี้) — ยังไม่ได้ทดสอบ manual ในเบราว์เซอร์จริง (ปุ่มเสียง/จังหวะเลื่อนอัตโนมัติ/reduced-motion/จังหวะ door-open transition)
- ตัดแพ็กเกจนี้เป็น **tag/release v1.17.0**
