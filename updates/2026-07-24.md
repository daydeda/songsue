## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **หน้าเลือกบ้านมีชุดสีประจำเฮาส์ให้ดูแล้ว** — ใต้คำบรรยายของแต่ละบ้านในหน้าธงประจำบ้าน (flag carousel) เพิ่มไอคอนธีมบ้าน + จุดสีชุดที่ใช้ในงานประจำปีของแต่ละบ้าน (Massfenrir, Chronokinesis, Ashkayn, Ancestral Incantation) ให้เห็นชัดว่าใส่ชุดโทนไหนไปงาน
- **ชื่อบ้าน (house name) ใช้ฟอนต์เฉพาะของแต่ละบ้านแล้ว** — เดิมทุกบ้านใช้ฟอนต์เดียวกัน ตอนนี้แต่ละบ้านมีลายเส้นตัวอักษรเป็นของตัวเองให้เข้ากับธีม
- **แก้บั๊กชื่อคณะ (faculty) บางบ้านตัดบรรทัดไม่สวย** — เดิมใช้ขนาดตัวอักษรคงที่ตามความกว้างจอ (clamp) ซึ่งบางชื่อภาษาไทยตัดขึ้นบรรทัดใหม่แบบไม่ตั้งใจ ตอนนี้ระบบย่อขนาดตัวอักษรอัตโนมัติให้พอดีบรรทัดเดียวเสมอ ไม่ว่าชื่อคณะจะยาวแค่ไหนหรือภาษาไหน

## ฝั่งทีม (technical changelog)

### เพิ่ม dress-code swatches และฟอนต์เฉพาะบ้าน (PR #37)

- **`src/components/home/houses-data.ts`** — เพิ่มฟิลด์ `dressCode: { colors: string[] } | null` ต่อบ้าน เก็บ hex code สีชุดทางการของแต่ละบ้าน
- **`src/components/home/house-fonts.ts` (ใหม่)** — โหลดฟอนต์ dafont.com 4 ตัว (Bazzotte, Starstruck, MirageGothic, FairyDustB) ผ่าน `next/font/local` (self-host ในโฟลเดอร์ `fonts/` เพราะ dafont ไม่มี CDN URL ที่เสถียร) แล้ว map เป็น `houseNameFontClassById` โดยคีย์ตาม `HouseInfo.id`
- **`src/components/home/SongsueLanding.tsx`** — เพิ่ม `houseDressCodeIconById` (ไอคอน lucide ต่อบ้าน) และ UI แสดงไอคอน+จุดสีใต้ caption; เพิ่ม hook `useFitOneLine` ที่วัดความกว้าง container จริงแล้วย่อ `font-size` ของชื่อคณะแบบ direct DOM style override (ไม่ re-render) จนพอดีบรรทัดเดียว แทนการพึ่ง `clamp()` แบบ static ตัวเดียวที่ไม่ครอบคลุมทุกความยาวชื่อคณะ TH/EN
- ตรวจสอบด้วย `npm run build` (ผ่าน) และ `npm run lint` (ไม่มี error/warning ใหม่ในไฟล์ที่แก้ — error เดิม 4-5 จุดใน `DoorCastle3D.tsx`/`SoftShadows.tsx` เป็นโค้ดเก่า ไม่เกี่ยวกับการแก้ไขนี้) — ยังไม่ได้ทดสอบ manual ในเบราว์เซอร์จริงสำหรับทั้ง 4 บ้านและทั้งสองภาษา
- ตัดแพ็กเกจนี้เป็น **tag/release v1.18.0**
