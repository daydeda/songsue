## ฝั่งนักศึกษา (สิ่งที่ user จะเห็น)

- **หน้าเลือกบ้านมีชุดสีประจำเฮาส์ให้ดูแล้ว** — ใต้คำบรรยายของแต่ละบ้านในหน้าธงประจำบ้าน (flag carousel) เพิ่มไอคอนธีมบ้าน + จุดสีชุดที่ใช้ในงานประจำปีของแต่ละบ้าน (Massfenrir, Chronokinesis, Ashkayn, Ancestral Incantation) ให้เห็นชัดว่าใส่ชุดโทนไหนไปงาน
- **ชื่อบ้าน (house name) ใช้ฟอนต์เฉพาะของแต่ละบ้านแล้ว** — เดิมทุกบ้านใช้ฟอนต์เดียวกัน ตอนนี้แต่ละบ้านมีลายเส้นตัวอักษรเป็นของตัวเองให้เข้ากับธีม
- **แก้บั๊กชื่อคณะ (faculty) บางบ้านตัดบรรทัดไม่สวย** — เดิมใช้ขนาดตัวอักษรคงที่ตามความกว้างจอ (clamp) ซึ่งบางชื่อภาษาไทยตัดขึ้นบรรทัดใหม่แบบไม่ตั้งใจ ตอนนี้ระบบย่อขนาดตัวอักษรอัตโนมัติให้พอดีบรรทัดเดียวเสมอ ไม่ว่าชื่อคณะจะยาวแค่ไหนหรือภาษาไหน
- **แก้บั๊กหัวข้อใหญ่หน้าแรก ("สองสื่อแบบศิลป์" / "Two Media in Arts") ล้นจอบนมือถือจอแคบ** (ยืนยันบั๊กบน iPhone 15 จริง) — เดิมตัวอักษรบังคับอยู่บรรทัดเดียวโดยไม่มีการย่อขนาด ทำให้ข้อความ (โดยเฉพาะเวอร์ชัน EN ที่ยาวกว่า) ล้นออกไปทั้งสองข้างของจอ ตอนนี้ย่อขนาดตัวอักษรอัตโนมัติให้พอดีจอเสมอเหมือนกับที่แก้ไว้กับชื่อคณะ
- **แก้บั๊กหน้าธงประจำบ้าน (House Flags) บางครั้งเลื่อนหน้าจอไม่ได้บนมือถือ** — ผู้ใช้รายงานว่าบางครั้งปัดจอลงไม่ได้ ต้อง refresh ถึงจะกลับมาเลื่อนได้ (และบางครั้ง refresh แล้วก็ยังไม่หาย) สาเหตุคือจังหวะที่ธงเปลี่ยนบ้านอัตโนมัติทุก 15 วินาทีไปชนกับตอนที่ผู้ใช้กำลังปัดธงอยู่พอดี ตอนนี้ระบบจะรอให้ปัดเสร็จก่อนค่อยเปลี่ยนบ้าน ไม่ชนกันอีก

## ฝั่งทีม (technical changelog)

### เพิ่ม dress-code swatches และฟอนต์เฉพาะบ้าน (PR #37)

- **`src/components/home/houses-data.ts`** — เพิ่มฟิลด์ `dressCode: { colors: string[] } | null` ต่อบ้าน เก็บ hex code สีชุดทางการของแต่ละบ้าน
- **`src/components/home/house-fonts.ts` (ใหม่)** — โหลดฟอนต์ dafont.com 4 ตัว (Bazzotte, Starstruck, MirageGothic, FairyDustB) ผ่าน `next/font/local` (self-host ในโฟลเดอร์ `fonts/` เพราะ dafont ไม่มี CDN URL ที่เสถียร) แล้ว map เป็น `houseNameFontClassById` โดยคีย์ตาม `HouseInfo.id`
- **`src/components/home/SongsueLanding.tsx`** — เพิ่ม `houseDressCodeIconById` (ไอคอน lucide ต่อบ้าน) และ UI แสดงไอคอน+จุดสีใต้ caption; เพิ่ม hook `useFitOneLine` ที่วัดความกว้าง container จริงแล้วย่อ `font-size` ของชื่อคณะแบบ direct DOM style override (ไม่ re-render) จนพอดีบรรทัดเดียว แทนการพึ่ง `clamp()` แบบ static ตัวเดียวที่ไม่ครอบคลุมทุกความยาวชื่อคณะ TH/EN
- ตรวจสอบด้วย `npm run build` (ผ่าน) และ `npm run lint` (ไม่มี error/warning ใหม่ในไฟล์ที่แก้ — error เดิม 4-5 จุดใน `DoorCastle3D.tsx`/`SoftShadows.tsx` เป็นโค้ดเก่า ไม่เกี่ยวกับการแก้ไขนี้) — ยังไม่ได้ทดสอบ manual ในเบราว์เซอร์จริงสำหรับทั้ง 4 บ้านและทั้งสองภาษา
- ตัดแพ็กเกจนี้เป็น **tag/release v1.18.0**

### แก้หัวข้อใหญ่หน้าแรกล้นจอบนมือถือจอแคบ (PR #39)

- **`src/components/home/SongsueLanding.tsx`** — hero `<h1>` เดิมใช้ `whiteSpace: nowrap` ร่วมกับ `clamp()` อิง `14vw` โดยไม่มีกลไกย่อขนาดเมื่อไม่พอ ทำให้ข้อความล้นออกทั้งสองข้างของจอบนมือถือจอแคบ (ยืนยันบน iPhone 15) โดยเฉพาะโหมด EN ที่ข้อความ "Two Media in Arts" ยาวกว่าเวอร์ชันไทย
- ใช้ hook `useFitOneLine` ตัวเดียวกับที่แก้ชื่อคณะใน flag carousel (ดูหัวข้อ PR #37 ด้านบน) มาย่อขนาดตัวอักษรของหัวข้อใหญ่ให้พอดีบรรทัดเดียวเสมอ พร้อมปรับ container ให้เป็น `w-full` ตลอดสาย เพื่อให้วัดความกว้างที่เหลือได้ถูกต้อง (ไม่งั้นจะวนเป็น circular เพราะ flex item ที่ `align-items: center` จะหดตัวตามเนื้อหาของตัวเอง)
- ตรวจสอบด้วย `npm run build` (ผ่าน) และ `npm run lint` (ไม่มี error/warning ใหม่) — ยังไม่ได้ยืนยันภาพจริงบนอุปกรณ์หลัง deploy เพราะบั๊กเดิมพบจากการทดสอบบนเว็บที่ deploy แล้ว ไม่ใช่ local dev
- ตัดแพ็กเกจนี้เป็น **tag/release v1.18.1**

### แก้บั๊ก House Flags carousel ชน mobile scroll ระหว่างปัด (PR #41)

- **`src/components/home/SongsueLanding.tsx`** — `FlagsCarousel` มี auto-advance timer เปลี่ยนบ้านทุก 15 วินาทีที่ทำงานเป็นอิสระจากการโต้ตอบของผู้ใช้ ในขณะที่ธงที่แสดงอยู่ก็รองรับ drag gesture ปัดเพื่อเปลี่ยนบ้านด้วย — ถ้า timer ทำงานพอดีตอนผู้ใช้กำลังปัด (นิ้วยังแตะจออยู่) `AnimatePresence` จะ unmount element ที่กำลังถูกลากอยู่ออกกลางคัน ซึ่งบน mobile ทำให้ Framer Motion ค้าง touch-handling ไว้ อ่านเหมือนหน้าเว็บเลื่อนไม่ได้จนกว่าจะ refresh (ตรงกับอาการที่รายงาน: เป็นบ้าง ไม่เป็นบ้าง refresh บางทีก็หาย บางทีก็ไม่หาย)
- เพิ่ม `isDraggingRef` set ใน `onDragStart`/`onDragEnd` ของธง แล้วให้ auto-advance timer เช็คค่านี้ก่อนเปลี่ยนบ้านทุกครั้ง (poll ทุก 500ms ถ้ากำลังลากอยู่) เพื่อไม่ให้ unmount element ระหว่างที่ยังลากค้างอยู่อีก
- ตรวจสอบด้วย `npm run build` (ผ่าน) และ `npm run lint` (ไม่มี error/warning ใหม่ในไฟล์ที่แก้) — ยังไม่ได้ยืนยัน manual retest บนมือถือจริงหลัง deploy (แนะนำให้ปัดธงไปมาต่อเนื่อง 15-20+ วินาทีเพื่อให้ auto-advance เกือบทำงานพอดีตอนปัด แล้วดูว่าเลื่อนจอยังติดอยู่ไหม)
- ตัดแพ็กเกจนี้เป็น **tag/release v1.18.2**
